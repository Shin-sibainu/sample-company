## 候補一覧

### 1. Git Worktrees × Claude Code — ソロ開発者が"小さなチーム"になる並列ワークフロー

- **日付**: 2026-04-19〜20（複数記事が集中）
- **ソース**: [Claude Code Worktrees: Parallel Sessions Without Conflicts](https://claudefa.st/blog/guide/development/worktree-guide) / [CLAUDE.md Productivity Stack - Botmonster](https://botmonster.com/posts/claude-md-productivity-stack-custom-commands-git-worktrees-agent-rules/) / [Parallel Development with ClaudeCode and Git Worktrees - Medium](https://medium.com/@ooi_yee_fei/parallel-ai-development-with-git-worktrees-f2524afc3e33)
- **カテゴリ**: コミュニティ / 技術動向
- **事実**: `claude -w` フラグで独立した git worktree を作成し、3〜5 セッションを並列実行するパターンが 2026 年の"標準ワークフロー"として定着しつつある。v2.1.50 でワークツリーの作成・管理・クリーンアップが CLI・Desktop 双方に組み込まれた。Notification フックと組み合わせることで、全セッションを手放しで走らせて「ブロックされたときだけ通知を受け取る」運用が可能。10〜15 並列まで実績あり。
- **考察**: 「一人で複数タスクを並列で回す」は読者が最も実感しやすい生産性向上の話。「ソロ開発者がチームになる」という表現は引用されやすくエンゲージメントが高い。Tips 型として worktree の命名規則・async フック設定など具体手順を入れると保存率が上がる。
- **注目度**: 高
- **推奨投稿型**: Tips型

### 2. Claude Code vs Codex — Reddit 500 人調査で浮かび上がった使い分け実態

- **日付**: 2026-04-18〜20
- **ソース**: [Claude Code vs Codex 2026 — What 500+ Reddit Developers Really Think - DEV Community](https://dev.to/_46ea277e677b888e0cd13/claude-code-vs-codex-2026-what-500-reddit-developers-really-think-31pb)
- **カテゴリ**: コミュニティ
- **事実**: DEV Community の記事が r/ClaudeAI（週間アクティブ 4,200 人）と r/Codex（同 1,200 人）のスレッドを横断分析。Claude Code はエージェント長時間タスク・コンテキスト保持・コードベース全体把握で優位。Codex は短いスニペット補完・VS Code 統合の滑らかさで好評。「どちらか一方」ではなく「場面で使い分ける」派が多数。
- **考察**: 「Claude Code と Codex どっちがいい？」は永遠に湧くトピック。Reddit 調査という一次データの裏付けがあるため議論型に適している。「エージェント長時間タスクは Claude Code」という主張で自社の文脈に引き込める。r/ClaudeCode のアクティブ数がライバルの 3.5 倍という数字もフックになる。
- **注目度**: 高
- **推奨投稿型**: 議論型

### 3. Hooks vs CLAUDE.md — 指示遵守率 100% vs 70% の差を知っていますか

- **日付**: 2026-04-17〜20（複数媒体で言及が重なる）
- **ソース**: [Claude Code power user tips | Claude Help Center](https://support.claude.com/en/articles/14554000-claude-code-power-user-tips) / [CLAUDE.md Productivity Stack - Botmonster](https://botmonster.com/posts/claude-md-productivity-stack-custom-commands-git-worketrees-agent-rules/) / [Claude Code Best Practices - rosmur.github.io](https://rosmur.github.io/claudecode-best-practices/)
- **カテゴリ**: 技術動向 / コミュニティ
- **事実**: 公式ドキュメントおよび複数のコミュニティ記事が「CLAUDE.md の指示は約 70% の確率でしか守られないが、フックは 100% 実行される」と明記。重要なガードレール（フォーマット強制・禁止コマンドチェック・コミット前テスト実行など）は CLAUDE.md ではなくフックに移すべきという指針が浸透しつつある。フックの設定は `.claude/settings.json` の `hooks` キーに記述する。
- **考察**: 「CLAUDE.md に何でも書けばいい」という誤解を崩すネタ。70% vs 100% という対比は覚えやすく拡散しやすい。「自分は CLAUDE.md に頼りすぎていた」という自己開示型で書くと共感を呼ぶ。実際の hooks 設定例を 1〜2 行添えるとすぐ試せる Tips になる。
- **注目度**: 高
- **推奨投稿型**: 解説型

### 4. Claude Code 劣化問題 — Reddit・Threads で「前より遅い/弱い」声が急増

- **日付**: 2026-04-17〜20（Threads 投稿が拡散中）
- **ソース**: [Threads - @that.map.guy.craig](https://www.threads.com/@that.map.guy.craig/post/DXHh89xFISa/) / [Claude Opus 4.7: What X and Reddit Users Are Saying - Botmonster](https://botmonster.com/posts/claude-opus-4-7-x-reddit-reception/)
- **カテゴリ**: コミュニティ
- **事実**: Threads と Reddit で「Claude Code が以前より応答が遅い・精度が落ちた」という声が急増。Opus 4.7 リリース前後からセッションキャップが数分で消費されるという報告も目立つ。一方で「自分は全く影響を受けていない」という反論も同数程度存在し、意見が二極化。Anthropic からの公式コメントは現時点で未確認。
- **考察**: 賛否が割れているトピックは議論型の鉄板。「あなたはどう感じる？」と問いかければコメントが集まる。Opus 4.7 の性能向上ベンチマーク（既投稿）と並べると「ベンチマークと実感のギャップ」という深い問いかけにもなる。Anthropic が応答しない点も火種。ただし事実確認が難しい部分があるため「コミュニティでこういう声が出ている」という立場で書くのが安全。
- **注目度**: 高
- **推奨投稿型**: 議論型

### 5. MCP × Claude Code 実用化 — Notion・Figma・DB をそのまま繋ぐ開発フロー

- **日付**: 2026-04-16〜20
- **ソース**: [5 Claude Code Agentic Workflow Patterns - MindStudio](https://www.mindstudio.ai/blog/claude-code-agentic-workflow-patterns) / [Claude Code Workflows and Best Practices 2026 - smart-webtech.com](https://smart-webtech.com/blog/claude-code-workflows-and-best-practices/)
- **カテゴリ**: 技術動向
- **事実**: `claude mcp add` コマンドで Notion・Figma・PostgreSQL などの外部ツールを Claude Code に接続し、イシュートラッカーの仕様を読み込みながらコーディングしたり、DB スキーマを参照しながらマイグレーションを書いたりするワークフローが普及。MCP サーバーの結果が大きい場合の永続化機能（larger MCP result persistence）も最新バージョンで対応済み。
- **考察**: 「Claude Code がコードだけ書くツール」という認識を壊すネタ。「Notion のチケットを読んでそのままコードに落とす」は多くのエンジニアが「それ便利では？」と感じる具体例。awesome-cc-oss（GitHub）でコミュニティ製 MCP 一覧も整備されておりソースが豊富。
- **注目度**: 中
- **推奨投稿型**: 解説型

### 6. Claude Code 利用コスト実態調査 — 月何万円かかっているか

- **日付**: 2026-04-15〜20（Reddit スレッド継続中）
- **ソース**: [Claude Code Reddit: What Developers Actually Use It For in 2026](https://www.aitooldiscovery.com/guides/claude-code-reddit) / [Claude Code Tips: 10 Real Productivity Workflows for 2026 - f22labs](https://www.f22labs.com/blogs/10-claude-code-productivity-tips-for-every-developer/)
- **カテゴリ**: コミュニティ
- **事実**: r/ClaudeAI・r/ClaudeCode で「月間 API コストがいくらか」のスレッドが定期的に上位に入る。報告値は月 $30〜$400 超まで幅広く、作業スタイルによる差が大きい。「Max プランにしたらコスト予測が立てやすくなった」という報告と、「Opus 4.7 xhigh モードで予想外の消費」という報告が並立。Caveman Mode や prompt caching 設定で大幅削減できたというコメントも多い。
- **考察**: コスト話題は誰にとっても関心が高い。具体的な金額が出ると「自分と比較したい」というコメントが集まりやすい。「あなたは月いくら使ってる？」という問いかけはシンプルで強力。Task Budgets（既出リサーチ済み）への誘導にもなる。
- **注目度**: 中
- **推奨投稿型**: 議論型

| # | トピック | 投稿型 | 注目度 |

### 選定理由

- **速報型（劣化問題）**: Opus 4.7 リリース直後のコミュニティ反応として鮮度が高い。賛否が割れているためエンゲージメントが見込める。Anthropic の公式応答待ちという"続報あり感"もある。
- **解説型（Hooks vs CLAUDE.md）**: 「70% vs 100%」という数字が明快で解説しやすい。公式ドキュメントに根拠があり信頼性も高い。CLAUDE.md ネタは過去に「開発OS思想」文脈で触れたが、フックとの比較という角度は未使用。
- **Tips型（Git Worktrees）**: 実行可能な手順が明確で保存・ブックマーク率が高い。`claude -w` というコマンドの具体性が強い。Skill/Commandとは別の並列化アプローチとして既出ネタとの差別化も明確。
- **議論型（vs Codex）**: 「どっちがいい？」という問いかけは回答コメントが集まりやすい。r/ClaudeCode のアクティブ数 3.5 倍という数字がフックになる。Anthropic エコシステムへの愛着を示しながら冷静に比較する立場が取りやすい。
