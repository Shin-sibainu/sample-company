### 2. Advisor Strategy — Haiku + Opus の組み合わせで単独 Sonnet より 85% 安くスコア 2 倍

- **日付**: 2026-04-09（パブリックベータ開始）
- **ソース**: [Advisor tool — Claude API Docs](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool) / [The Advisor Strategy — Anthropic Blog](https://claude.com/blog/the-advisor-strategy) / [Anthropic Advisor Strategy: Smarter AI Agents](https://www.buildfastwithai.com/blogs/anthropic-advisor-strategy-claude-api) / [Medium解説](https://medium.com/@ai_93276/the-advisor-strategy-how-anthropics-new-pattern-delivers-opus-level-agents-at-sonnet-prices-933510b21200)
- **カテゴリ**: 公式発表（APIアップデート）
- **事実**: 4/9 に公開ベータ入りした「Advisor Tool（`advisor-tool-2026-03-01` ベータヘッダー）」は、安価な executor モデル（Sonnet/Haiku）が長時間タスクを処理しながら、行き詰まった局面でのみ高性能な advisor モデル（Opus）に相談する設計。Sonnet + Opus advisor は SWE-bench Multilingual で単独 Sonnet より +2.7pt 向上しながらコストは 11.9% 削減。Haiku + Opus advisor では Haiku 単独スコアを 19.7% → 41.2% と 2 倍以上に引き上げつつ、単独 Sonnet 比でコストを 85% 削減。
- **考察**: 「どのモデルをデフォルトにするか」という従来の二択から「役割分担」という概念への転換点。Claude Code の内部でも同様の発想（サブエージェントが Opus に相談する形）が組み込まれる可能性がある。「コストと性能のトレードオフ問題」に対するアーキテクチャ的回答として解説しやすい。数値（85%コスト削減・スコア2倍）が具体的でバズりやすい。
- **注目度**: 高
- **推奨投稿型**: 解説型

### 3. ant CLI — Claude API を YAML で管理するコマンドラインクライアント正式リリース

- **日付**: 2026-04-08
- **ソース**: [Claude Platform Release Notes — April 8, 2026](https://platform.claude.com/docs/en/release-notes/overview) / [Claude Platform API CLI Reference](https://platform.claude.com/docs/en/api/sdks/cli)
- **カテゴリ**: 公式発表
- **事実**: Anthropic が `ant` CLI をリリース。Claude API への直接操作、API リソース（エージェント設定・プロンプト・ツール定義など）の YAML ファイルによるバージョン管理、Claude Code とのネイティブ統合が特徴。Managed Agents のセッション管理や設定の Git 管理が `ant` コマンド一本で完結する。`npm install -g @anthropic/ant` でインストール可能。
- **考察**: Claude Code ユーザーにとって「プロンプトや設定を YAML でコードと一緒に管理できる」は実務的に大きい。インフラのコード化（IaC）と同じ発想を AI エージェント設定に持ち込んだもの。`CLAUDE.md` と組み合わせた設定管理の進化形として Tips 投稿に仕立てやすい。Managed Agents ダッシュボード（既投稿）と組み合わせると深みが出るが、独立した Tips としても十分成立する。
- **注目度**: 中
- **推奨投稿型**: Tips型

### 7. git worktree × Claude Code — 並列 AI 開発で生産性 18% 向上の報告

- **日付**: 2026-04 継続（複数 Medium・DEV.to 記事が相次いで公開中）
- **ソース**: [Mastering Git Worktrees with Claude Code — Medium](https://medium.com/@dtunai/mastering-git-worktrees-with-claude-code-for-parallel-development-workflow-41dc91e645fe) / [Claude Code Common Workflows — Docs](https://code.claude.com/docs/en/common-workflows) / [MindStudio: Git Worktree Pattern](https://www.mindstudio.ai/blog/what-is-the-claude-code-git-worktree-pattern-parallel-feature-branches) / [DEV Community: Running Multiple Claude Code Sessions in Parallel](https://dev.to/datadeer/part-2-running-multiple-claude-code-sessions-in-parallel-with-git-worktree-165i)
- **カテゴリ**: コミュニティ / 技術動向
- **事実**: `claude -w` フラグで git worktree を使い、複数の Claude Code セッションを並列実行するワークフローがコミュニティで広く共有されている。各 worktree は `.claude/worktrees/` 以下に独立ブランチとして作成され、コンテキスト干渉なく同時進行できる。Anthropic 社内では一人で 10〜15 並列セッションを走らせる例もあり、チームで最大 18% のスループット向上が報告されている。Desktop App のマルチセッション刷新（既投稿）と組み合わせると理解が深まる切り口。
- **考察**: Desktop App のマルチセッション機能（4/17 既投稿）と同じ「並列」というテーマだが、こちらは CLI + git workflow という実装寄りの話。「1 人でチームのように動く」というコンセプトは日本の個人開発者・フリーランスに刺さる。具体的な CLI コマンド（`claude -w feature/xxx`）を示すと実行可能性が高い。
- **注目度**: 中
- **推奨投稿型**: Tips型
