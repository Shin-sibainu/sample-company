## 候補一覧

### 1. 使用量ドレイン問題の長期化 — Anthropic が「最優先対応中」を認めながら約4週間未解決

- **日付**: 2026-03-23（発生）〜 2026-04-13（The Register 続報）
- **ソース**: [Anthropic admits Claude Code quotas running out too fast - The Register](https://www.theregister.com/2026/03/31/anthropic_claude_code_limits/) / [Anthropic: Claude quota drain not caused by cache tweaks - The Register](https://www.theregister.com/2026/04/13/claude_code_cache_confusion/) / [GitHub Issue #41930](https://github.com/anthropics/claude-code/issues/41930)
- **カテゴリ**: コミュニティ / 技術動向
- **事実**: 2026-03-23 以降、Pro・Max 全プランで使用量が異常に早く消費される問題が多発。Max 20× の 5 時間ウィンドウが 19 分で枯渇する事例も報告。Anthropic は「way faster than expected、最優先で調査中」と公式に認めたが、4/13 時点でも完全解決の公式アナウンスなし。GitHub Issue では 4 つの根本原因が特定されており、プロンプトキャッシュのバグ（コスト 10〜20 倍化）・セッション再開時の全コンテキスト再処理・意図的なピーク時スロットリング・プロモーション終了が重複して発生。
- **考察**: 「有料サービスが約 4 週間にわたり正常動作しないのにブログ投稿もメール通知もない」という透明性の欠如が、コミュニティの不満を増幅させた。Claude Code を業務で使う層への信頼性問題として重要。「なぜ使用量が減るのか？自分でできる診断方法」というTips 方向でも、「Anthropicの危機対応に何が欠けていたか」という議論方向でも使えるトピック。
- **注目度**: 高
- **推奨投稿型**: 議論型

### 2. /powerup — ターミナルを離れず完結する 18 レッスンのインタラクティブ学習コマンド

- **日付**: 2026-04-01（v2.1.90 リリース）
- **ソース**: [Claude Code /powerup Command Guide 2026 - Claude Lab](https://claudelab.net/en/articles/claude-code/claude-code-powerup-command-guide) / [Claude Code April 2026 Update: /powerup, MCP 500K, Session Stability - Daily 1 Bite](https://daily1bite.com/en/blog/ai-tutorial/claude-code-april-2026-update) / [Claude Code Tutorial: /powerup Interactive Lessons Guide - claudefa.st](https://claudefa.st/blog/guide/mechanics/claude-powerup)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: v2.1.90（2026-04-01 リリース）で `/powerup` コマンドが追加。初級・中級・上級の 3 段階・18 レッスンが Claude Code インストールパッケージにバンドルされており、オフラインでも実行可能。アニメーション付きデモをターミナル内で再生しながら主要機能を体験的に習得できる。既存ユーザーも忘れがちな機能の再学習に使える構成。
- **考察**: Claude Code の習得コストは「何ができるか知らない」ことで生まれることが多い。`/powerup` はオンボーディングの自己解決手段として機能し、企業導入時のコスト削減につながる。「Cursor や GitHub Copilot から乗り換えた人が最初に走らせるべきコマンド」という切り口で Tips 型投稿に最適。日本語での紹介はまだ少ない。
- **注目度**: 中
- **推奨投稿型**: Tips型

### 3. MCP 500K 文字対応 + Result Persistence — 大規模コードベース全体をコンテキストへ

- **日付**: 2026-04-07 頃（v2.1.91 リリース）
- **ソース**: [Master 5 New Features of Claude Code v2.1.92 - Apiyi.com Blog](https://help.apiyi.com/en/claude-code-v2-1-92-mcp-persistence-powerup-tutorial-en.html) / [I Trained Claude Code on Our Entire Codebase Using MCP - Medium](https://medium.com/engineering-playbook/i-trained-claude-code-on-our-entire-codebase-using-mcp-c709263094ca) / [Claude Code April 2026 Update - Daily 1 Bite](https://daily1bite.com/en/blog/ai-tutorial/claude-code-april-2026-update)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: v2.1.91 で MCP ツール呼び出し結果の最大サイズが 500,000 文字まで拡張（`_meta["anthropic/maxResultSizeChars"]: 500000` で指定）。加えて MCP Result Persistence により、DB クエリ・ログファイル・大規模ファイル読み込みの結果をセッション間で保持できるようになった。以前は大型ファイルや長いログが MCP 経由で返ってきた場合に切り捨てられていたが、それが解消された。Medium の実例記事では「コードベース全体を MCP で学習させたところ、テックリードより良いアーキテクチャ判断をするようになった」という報告も話題になっている。
- **考察**: 「Claude Code + MCP」を本番レベルで使おうとして「結果が切り捨てられる」壁にぶつかっていた層に直接刺さる改善。Managed Agents・Routines と組み合わせると「全社コードベースを常時認識するエージェント」が現実的になる。技術詳細を省いて「コードベース全体を読ませる方法」という実用 Tips として投稿すると広く届く。
- **注目度**: 中
- **推奨投稿型**: Tips型

### 4. Hooks defer 機能 + 26 ライフサイクルイベント — 外部承認ゲートを CI/CD に組み込む

- **日付**: 2026-04-07（v2.1.89 で defer 追加）〜 2026-04-18（v2.1.114 時点で 26 イベント）
- **ソース**: [Hooks reference - Claude Code Docs](https://code.claude.com/docs/en/hooks) / [Claude Code Hooks: All 12 Events with Examples - Pixelmojo](https://www.pixelmojo.io/blogs/claude-code-hooks-production-quality-ci-cd-patterns) / [Claude Code Hooks: Why Each of My 95 Hooks Exists - blakecrosley.com](https://blakecrosley.com/blog/claude-code-hooks)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: v2.1.89 で `PreToolUse` フックに `defer` 値が追加。`-p`（print モード）セッションはツール呼び出し直前で一時停止し `deferred_tool_use` ペイロードを出力して終了、その後 `--resume` で再開できる。これにより SDK アプリや独自 UI 側で「このツール呼び出しを許可するか」を人間またはシステムが判断する承認ゲートを外部に持てるようになった。v2.1.114（2026-04-18）時点で Claude Code が公開するフックイベント数は 26 に到達。条件付きフィルタリング（複合コマンド `ls && git push` や `FOO=bar git push` への正確なマッチ）も修正済み。
- **考察**: セキュリティ審査や変更管理プロセスが必要なエンタープライズ環境で「AI エージェントの行動を人間が制御できるか」という問いへの明確な答え。Hooks はプロンプト指示と異なり決定論的に実行されるため、監査証跡を残したい企業用途に直結する。「エージェントを本番環境で走らせる前に必ず設定すべき Hooks 3 選」のような実用 Tips 投稿に活用できる。
- **注目度**: 中
- **推奨投稿型**: 解説型

### 5. OS CA 証明書デフォルト信頼 — 企業 TLS プロキシがゼロ設定で動くようになった

- **日付**: 2026-04-10（v2.1.101 リリース）
- **ソース**: [Enterprise network configuration - Claude Code Docs](https://code.claude.com/docs/en/network-config) / [Claude Code v2.1.101: Enhanced Team Collaboration and Enterprise Support - ClaudeWorld](https://claude-world.com/articles/claude-code-21101-release/) / [Decoding the Claude Code April 2026 Changelog - Apiyi.com](https://help.apiyi.com/en/claude-code-changelog-2026-april-updates-en.html)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: v2.1.101 から Claude Code がデフォルトで OS の CA 証明書ストアを信頼するように変更。CrowdStrike Falcon・Zscaler などの TLS インスペクションプロキシが、ルート証明書を OS 側に導入済みであれば Claude Code 側の追加設定不要で動作するようになった。以前は `NODE_EXTRA_CA_CERTS` や `SSL_CERT_FILE` などの環境変数を手動設定する必要があり、企業 IT 部門のサポートコストになっていた。バンドル Mozilla CA のみに戻したい場合は `CLAUDE_CODE_CERT_STORE=bundled` で上書き可能。
- **考察**: 企業・チーム導入時の「社内ネットワークで接続できない」問題は最大の初期離脱要因の一つ。これが自動解決されることで、エンタープライズでの採用ハードルが大幅に下がる。IT 担当者・CTO・DevOps エンジニア向けに「Claude Code を社内展開する際のチェックリスト」という文脈で言及すると実用性が高い。地味だが企業ユーザーには刺さるネタ。
- **注目度**: 中
- **推奨投稿型**: Tips型

### 6. claude.ai でのサービス障害 — Tax Day（4/13〜15）に集中発生

- **日付**: 2026-04-13〜15
- **ソース**: [Is Claude down? There goes my day - SF Standard](https://sfstandard.com/2026/04/15/is-claude-down-again-tax-day/) / [Claude Status](https://status.claude.com/)
- **カテゴリ**: コミュニティ / 技術動向
- **事実**: 2026-04-13 に claude.ai・API・Claude Code で HTTP 500 エラーが断続的に発生。米国の確定申告締切日（Tax Day、4/15）に向けて業務利用が集中したタイミングと重なり、SF Standard など地域メディアが「また落ちた」と報じた。使用量ドレイン問題（候補 1）と時期が重複しており、ユーザーの不満がさらに増幅。
- **考察**: 「AI ツールへの業務依存が高まった結果、障害が社会問題化する」という新しい構図を示す事例。Claude Code を業務フローに組み込む際のリスク管理（オフライン時の代替手順整備など）についての議論喚起に使える。単純な障害報告ではなく「AI 依存リスクをどう管理するか」という問いかけ投稿として機能する。
- **注目度**: 中
- **推奨投稿型**: 議論型

- **注目度**: 中

## おすすめ4件（投稿用）

上記から最もX投稿に適した4件を選び、推奨する投稿型を割り当てる。

| #   | トピック | 投稿型 | 注目度 |
| --- | -------- | ------ | ------ |
| 1   | 使用量ドレイン問題の長期化 — 4週間未解決・Anthropicの透明性問題 | 議論型 | 高 |
| 2   | Claude Code System Prompts 公開リポジトリ — 内部思考を読んでプロンプト設計に活かす | 解説型 | 中 |
| 3   | /powerup — ターミナル離れず 18 レッスン完結のオンボーディングコマンド | Tips型 | 中 |

### 選定理由

- **議論型（使用量ドレイン）**: 「最優先対応中」と言いながら約4週間改善アナウンスなしというギャップは、AI ツール依存ユーザーの共感を強く引き出す。「あなたも体験した？」の問いかけでエンゲージメントが見込める。The Register が 2 度報じた点でも話題性が継続している。
- **解説型（System Prompts）**: 4/18 更新で鮮度が高い。「Claude Code の内部を知ってプロンプト力を上げる」という直接的な実用価値があり、リンク付き解説投稿としてサードパーティソースへの誘導が自然にできる。
- **Tips型（/powerup）**: 日本語での紹介がまだ少ない機能。「知らないと損」系の Tips として拡散しやすく、Claude Code を使い始めた層に刺さる入門ネタとして機能する。
- **解説型（Hooks defer）**: セキュリティ・ガバナンス文脈で企業導入層に訴求できる技術トピック。「エージェントを本番で使う前にやること」系の解説投稿と組み合わせやすい。
