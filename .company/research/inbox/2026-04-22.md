### 1. Claude Mythos Preview — 非公開の次世代モデルとProject Glasswing

- **日付**: 2026-04-07
- **ソース**: [Anthropic Releases Claude Mythos Preview with Cybersecurity Capabilities but Withholds Public Access - InfoQ](https://www.infoq.com/news/2026/04/anthropic-claude-mythos/) / [Axios](https://www.axios.com/2026/04/16/anthropic-claude-opus-model-mythos) / [Fortune](https://fortune.com/2026/04/07/anthropic-claude-mythos-model-project-glasswing-cybersecurity/)
- **カテゴリ**: 公式発表
- **事実**: Anthropic が4/7にClaude Mythos Previewを発表。一般公開はせず、Amazon Web Services・Apple・Cisco・CrowdStrike・Google・Nvidia等の11社で構成される「Project Glasswing」に限定提供。主な能力はサイバーセキュリティ特化で、主要OSとブラウザ全種のゼロデイ脆弱性を自律発見・悪用できるレベルに達しているとされる。最古の発見はOpenBSDの27年前の脆弱性。Claude Code with Mythos Previewはコードから脆弱性を仮説立て、実行して検証、PoC付きバグレポートを出力できる。
- **考察**: 「最強モデルを公開しない」という異例の判断がAnthropicの安全性方針を体現している。Opus 4.7がGA（既出）の裏でMythosが存在することは、「Claudeの能力の天井はどこか」という議論を呼ぶ。Claude Codeユーザーにとっては「自分たちが使えない最強のコーディングAI」というフラストレーションポイントにもなる。
- **注目度**: 高
- **推奨投稿型**: 議論型

### 2. Agent Teams — 複数のClaude Codeインスタンスが協調するSwarmモード

- **日付**: 2026-02-05（初期リリース）、2026-04-18（v2.1.114でクラッシュ修正）
- **ソース**: [Orchestrate teams of Claude Code sessions - Claude Code Docs](https://code.claude.com/docs/en/agent-teams) / [Claude Code Agent Teams: Setup & Usage Guide 2026 - claudefa.st](https://claudefa.st/blog/guide/agents/agent-teams) / [From Tasks to Swarms: Agent Teams in Claude Code - alexop.dev](https://alexop.dev/posts/from-tasks-to-swarms-agent-teams-in-claude-code/)
- **カテゴリ**: 公式発表（実験的機能）
- **事実**: `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` で有効化できる実験的機能。1つのセッションがチームリードとなり、複数のClaude Codeインスタンス（チームメイト）を起動して並列作業させる。チームメイト同士がメールボックス経由で直接通信・共有タスクリストで自律協調できる点がサブエージェントとの根本的な違い。Shift+Downでチームメイトを切り替え、tmux/iTerm2でペイン分割表示も可能。TeammateIdle/TaskCreated/TaskCompletedフックで品質ゲートも設定できる。v2.1.114（4/18）でチームメイトのパーミッションダイアログクラッシュを修正済み。
- **考察**: サブエージェントとの使い分けは「エージェント同士が対話する必要があるか否か」。競合仮説の並列検証・フロントエンド/バックエンド/テスト横断の同時開発が典型ユースケース。トークンコストが線形スケールする点は注意だが、「AIのスウォームで並列開発」という体験は未体験者への訴求力が高い。
- **注目度**: 高
- **推奨投稿型**: 解説型

### 3. NO_FLICKERモード＋/focusコマンド — ターミナルのちらつきを根絶する新レンダラー

- **日付**: 2026-03-31（NO_FLICKER初期リリース）、v2.1.97でFocus View追加
- **ソース**: [Boris Cherny on X](https://x.com/bcherny/status/2039421575422980329) / [Fullscreen rendering - Claude Code Docs](https://code.claude.com/docs/en/fullscreen) / [Claude Code v2.1.97: focus view, live status refresh - Buttondown](https://buttondown.com/claudecode/archive/claude-code-v2197-focus-view-live-status-refresh/) / [Anthropic Fixes Claude Code Terminal Flickering with NO_FLICKER - piunikaweb](https://piunikaweb.com/2026/04/02/anthropic-no-flicker-mode-claude-code/)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: `CLAUDE_CODE_NO_FLICKER=1`（v2.1.110以前）または`/tui fullscreen`コマンドでオルタネートスクリーンバッファを使った新レンダラーが有効化できる。長いセッションでもメモリ・CPU使用量が一定に保たれ、マウス操作も可能になる。v2.1.97で追加された`/focus`コマンド（Ctrl+Oに変更）はNO_FLICKERモード時にプロンプト・ツール要約・最終レスポンスのみ表示するFocus Viewに切り替え、長時間作業の集中を助ける。Write toolのdiff計算も60%高速化（タブ・$記号を含む大ファイル対象）。トレードオフとして、ネイティブのCmd+F検索とコピーペーストは非対応（代わりにCtrl+O→/でトランスクリプト検索）。
- **考察**: /tui fullscreenは4/17投稿済みだが、NO_FLICKERの環境変数による細かい設定・Write tool 60%高速化・CLAUDE_CODE_DISABLE_MOUSE=1の組み合わせは別トピックとして成立する。長時間エージェント実行中にターミナルがチラつく問題は多くのユーザーが経験している実用的な話題。
- **注目度**: 高
- **推奨投稿型**: Tips型

### 5. sandbox.network.deniedDomains — v2.1.113で追加されたセキュリティ細粒度設定

- **日付**: 2026-04-17（v2.1.113）
- **ソース**: [Releases · anthropics/claude-code - GitHub](https://github.com/anthropics/claude-code/releases) / [Changelog - Claude Code Docs](https://code.claude.com/docs/en/changelog)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: v2.1.113で`sandbox.network.deniedDomains`設定が追加された。これにより`allowedDomains`のワイルドカードで許可された範囲内でも特定ドメインだけをブロックできるようになった。例えば`*.example.com`を許可しつつ`internal.example.com`は遮断するという設定が可能。同バージョンではWindows向けにPowerShellツールのプログレッシブロールアウト（`CLAUDE_CODE_USE_POWERSHELL_TOOL`でオプトイン/アウト）も追加。
- **考察**: 企業環境でClaude Codeを本番運用する際のネットワーク制御が細かくなった。Managed Agents（既出）やRoutines（既出）と組み合わせた本格運用に向けて、セキュリティポリシーを満たしやすくなる。エンタープライズ採用の障壁を下げるネタとして実用的。
- **注目度**: 中
- **推奨投稿型**: Tips型
