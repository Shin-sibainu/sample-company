### 1. Claude Code v2.1.116 — 大型セッションの /resume が最大67%高速化

- **日付**: 2026-04-20
- **ソース**: [What's new - Claude Code Docs](https://code.claude.com/docs/en/changelog)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: v2.1.116（4/20リリース）で、40MB超の大型セッションに対する `/resume` が最大67%高速化。MCPの起動速度も改善（deferred `resources/templates/list`）。「thinking spinner」がインライン進捗表示（"still thinking" / "thinking more" / "almost done thinking"）に変わり、長時間タスクの進行が可視化される。`/doctor` がClaudeの応答中にも開けるようになり、`/config` 検索がオプション値にも対応。
- **考察**: 「Claudeが止まっているのか動いているのか分からない」「大型プロジェクトでの再開が遅い」という実務上の痛点を直撃するQoL改善。Thinking spinnerのインライン表示は開発者の体験向上に直結し、スクリーンショット付きのTips投稿に最適。
- **注目度**: 高
- **推奨投稿型**: Tips型

### 2. Claude Code ネイティブ VS Code 拡張機能（ベータ）— Sonnet 4.5 標準搭載

- **日付**: 2026-04-18〜20（公式ブログ "Enabling Claude Code to work more autonomously"）
- **ソース**: [Enabling Claude Code to work more autonomously - Anthropic](https://www.anthropic.com/news/enabling-claude-code-to-work-more-autonomously) / [Use Claude Code in VS Code - Claude Code Docs](https://code.claude.com/docs/en/vs-code)
- **カテゴリ**: 公式発表
- **事実**: Anthropicが公式VS Code拡張機能（ベータ）をリリース。Claude CodeをターミナルなしにVS Code内の専用サイドバーパネルから直接呼び出し、インラインdiffでリアルタイム変更確認が可能になった。デフォルトモデルがClaude Sonnet 4.5に更新され、ターミナルインターフェースもv2.0に刷新。チェックポイント機能（Escを2回押すか `/rewind` で任意のコード状態に即時巻き戻し）も同時提供。
- **考察**: 「Claude Codeを使いたいがターミナルに慣れていない」VS Codeユーザー層への大きな訴求。JetBrains・Xcode等でのGitHub Copilot統合が先行していた中、Anthropic公式対応は競合との差別化軸になる。「IDE派 vs ターミナル派」の議論も巻き起こせる。
- **注目度**: 高
- **推奨投稿型**: 速報型

- **考察**: 同日リリースは明らかに意識した競合対抗。Claude Code vs Codex の機能比較議論はコミュニティで加熱中。「コーディングAIの戦争は2026年に本格化した」という構図で議論型投稿に最適。Anthropicが勝っているポイントを具体的に挙げると建設的なエンゲージメントが期待できる。
- **注目度**: 高
- **推奨投稿型**: 議論型

### 4. /loop コマンド — 最大3日間のローカル定期タスク自動化

- **日付**: 2026-04-15〜（Week 15リリース以降継続的に話題）
- **ソース**: [What's new - Claude Code Docs](https://code.claude.com/docs/en/whats-new) / [Claude Code Tips: 10 Real Productivity Workflows for 2026](https://www.f22labs.com/blogs/10-claude-code-productivity-tips-for-every-developer/)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: `/loop` コマンドがローカル環境で最大3日間の繰り返しタスクをスケジュール実行できる機能として注目されている。インターバルを省略すると自動ペース調整。活用例：「レビューコメントへの自動対応」「PRを自動的にプッシュしてフィードバックを取得」。v2.1.113ではEscで保留中のwakeupをキャンセルできるよう改善。
- **考察**: Routines（クラウド実行・既出）の「ローカル版」に相当し、Pro以外のユーザーでも使える点が重要。「毎日の繰り返し作業をAIに任せる」という方向性は日本語コミュニティでも関心が高い。具体的なユースケース（テスト自動実行・PR管理など）で解説型ポストにも転用可能。
- **注目度**: 高
- **推奨投稿型**: Tips型

### 5. Context Engineering — 2026年の最大生産性レバー

- **日付**: 2026-04-18〜20（複数コミュニティ記事が集中）
- **ソース**: [Claude Code: Workflows and Best Practices 2026](https://smart-webtech.com/blog/claude-code-workflows-and-best-practices/) / [5 Claude Code Agentic Workflow Patterns - MindStudio](https://www.mindstudio.ai/blog/claude-code-agentic-workflow-patterns) / [Claude best practices 2026: the complete power user guide](https://www.the-ai-corner.com/p/claude-best-practices-power-user-guide-2026)
- **カテゴリ**: 技術動向
- **事実**: 「Context Engineering（コンテキストエンジニアリング）」がPrompt Engineeringに代わる2026年の最重要スキルとして複数コミュニティ記事で言及が急増。Anthropicの内部テストでは「ガイドなし実行は成功率33%」に対し、コンテキストを適切に設計すると大幅に向上するとされる。CLAUDE.mdの設計、Plan Modeの活用、ワークツリー並列実行（3〜5セッション）がキーパターンとして挙げられている。
- **考察**: 「モデルのせいではなくコンテキストの設計が問題」というメッセージは多くの開発者に刺さる。日本語での体系的な解説記事はまだ少ない。「プロンプト職人からコンテキスト設計者へ」という切り口で解説型投稿を作ると差別化できる。
- **注目度**: 高
- **推奨投稿型**: 解説型

### 6. v2.1.113 ネイティブバイナリ移行 — 起動速度・セキュリティが向上

- **日付**: 2026-04-17
- **ソース**: [Claude Code Changelog - code.claude.com](https://code.claude.com/docs/en/changelog)
- **カテゴリ**: 公式発表（技術）
- **事実**: v2.1.113でCLIがバンドルされたJavaScriptの代わりにネイティブのClaude Codeバイナリを起動するよう変更。`sandbox.network.deniedDomains` 設定で特定ドメインのネットワークアクセスをブロック可能に。Ctrl+A/Eが複数行入力の論理行頭・行末に対応、長いURLが折り返し後もクリッカブルを維持するなどUX改善も含む。
- **考察**: ネイティブバイナリ移行は起動速度だけでなくセキュリティ上の意味が大きい。「deniedDomains設定でサンドボックス環境を厳密に制御できる」はエンタープライズ・チーム利用における重要な訴求点。セキュリティ重視の開発者層に刺さるTips型として活用できる。
- **注目度**: 中
- **推奨投稿型**: Tips型

### 7. Anthropic model migration — 4/23に Enterprise デフォルトが Opus 4.7 へ移行

- **日付**: 2026-04-20（告知確認）
- **ソース**: [Models overview - Claude API Docs](https://platform.claude.com/docs/en/about-claude/models/overview) / [Migration guide - Claude API Docs](https://platform.claude.com/docs/en/about-claude/models/migration-guide)
- **カテゴリ**: 公式発表
- **事実**: 2026-04-23より、Anthropic APIのEnterprise従量課金ユーザーのデフォルトモデルがOpus 4.7に切り替わる。APIでは `opus` がOpus 4.7、`sonnet` がSonnet 4.6に解決される。Bedrock・Vertex・Foundryでは `opus` がOpus 4.6、`sonnet` がSonnet 4.5のまま。Sonnet 4.5は引き続きサポートされ、EOL日は未発表。
- **考察**: 既存API利用者にとって影響が大きい告知。「モデル名でエイリアスを使っているコードは動作が変わる可能性がある」という実務的な警告として速報型に使える。移行ガイドのURLとセットにすると有益度が高い。
- **注目度**: 中
- **推奨投稿型**: 速報型

### 8. /tui フルスクリーン + Auto-scroll 無効化オプション

- **日付**: 2026-04-15（v2.1.110）
- **ソース**: [Claude Code Changelog - code.claude.com](https://code.claude.com/docs/en/changelog)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: v2.1.110で `/tui` コマンドおよび `tui` 設定がフリッカーなしのフルスクリーンレンダリングを提供。`autoScrollEnabled` 設定でフルスクリーン中の自動スクロールを無効化可能。Ctrl+Oで通常/詳細トランスクリプト表示を切り替え。Push通知ツールが追加され「Claudeが判断したときにプッシュ」設定が利用可能に。
- **考察**: フルスクリーンTUIは長時間タスク実行中の視認性を大きく改善する。Remote ControlのPush通知と組み合わせると「タスクを任せて離席→完了通知を受け取る」ワークフローが完成する。使い方のデモとしてTips型投稿に適している。
- **注目度**: 中
- **推奨投稿型**: Tips型
