### 1. `gh skill` — GitHub CLI がAgent Skillsのパッケージマネージャーになった

- **日付**: 2026-04-16
- **ソース**: [Manage agent skills with GitHub CLI - GitHub Changelog](https://github.blog/changelog/2026-04-16-manage-agent-skills-with-github-cli/)
- **カテゴリ**: 公式発表
- **事実**: GitHub CLI v2.90.0 に `gh skill` コマンドが追加（パブリックプレビュー）。`gh skill install github/awesome-copilot documentation-writer --agent claude-code --scope user` のように、GitHubリポジトリからAgent Skillsを発見・インストール・管理・公開できるパッケージマネージャーとして機能する。スキルはタグ+gitのtree SHAで管理されるため、`gh skill update` で実際の内容変化のみ検出できる。バージョンピン留め（`--pin v1.2.0`）も可能。Claude Code・Codex・Cursor・Copilot・Gemini CLIなど複数エージェント間でスキルを共有できるAgent Skills仕様に準拠。セキュリティ注意点として「スキルはGitHubが検証しておらずプロンプトインジェクションの可能性あり、`gh skill preview` でインストール前確認を推奨」と明記されている。
- **考察**: Skills管理のパッケージマネージャー化は、npmやHomebrewと同じ流れを「エージェントの振る舞い」に適用した歴史的なステップ。Claude Code単独ではなくエコシステム全体の標準化という意味で注目度が高い。「使い捨てプロンプトからバージョン管理されたSkillへ」という開発者の意識変化を促す速報ネタ。
- **注目度**: 高
- **推奨投稿型**: 速報型

- **カテゴリ**: 公式発表（機能追加）
- **事実**: v2.1.90で追加された `/powerup` コマンドは、ターミナル内でClaude Codeの使い方をアニメーションデモ付きで学べる公式インタラクティブ学習システム。18本のレッスンが収録され、各レッスンは「解説テキスト + コードスニペット + アニメーション」の構成。完了済みレッスンはチェックマークで管理できる。全プラン（Free含む）で追加費用なし。「ドキュメントを読みに外部サイトへ飛ぶ必要がない」を明確なゴールとして設計されており、VS Codeのインタラクティブチュートリアルに相当するポジションを担う。

### 5. PowerShell Tool — WindowsネイティブのClaude Code開発環境がopt-inで解禁

- **日付**: 2026-04-15（v2.1.111）
- **ソース**: [Claude Code PowerShell Tool: Complete Guide for Native Windows Development - claudelab.net](https://claudelab.net/en/articles/claude-code/claude-code-powershell-tool-windows-guide) / [Claude Code integrates a new PowerShell tool - WIIP](https://wiip.fr/en/blog/claude-code-powershell-tool) / [Changelog - Claude Code Docs](https://code.claude.com/docs/en/changelog)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: v2.1.111でPowerShell ToolがWindowsユーザー向けにopt-inプレビューとして追加（`CLAUDE_CODE_USE_POWERSHELL_TOOL=1`）。LinuxとmacOSは`CLAUDE_CODE_USE_POWERSHELL_TOOL=1`で有効化できる。これまでWSL（Windows Subsystem for Linux）経由でGit Bashコマンドを介在させていた構造を廃止し、`pwsh.exe`または`powershell.exe`を直接スポーンする。パス変換エラー（`C:\` vs `/c/`）、POSIXフラグの失敗、エンコーディングミスマッチという3大Windows固有の問題を解消。ファイルシステム操作・レジストリアクセス・Azure/Active Directory統合がネイティブに動作する。正式ローンチは2026年5月頃を予定。
- **考察**: Windowsユーザーへの訴求力が高い実用Tips。「WSL前提だったClaude CodeがWindowsファーストに近づいている」という文脈はエンタープライズ・法人採用を後押しする。日本のWindowsユーザーは多く、差別化できる日本語コンテンツになりやすい。
- **注目度**: 中
- **推奨投稿型**: Tips型

### 7. `sandbox.network.deniedDomains` — ワイルドカード許可下でも特定ドメインをブロック

- **日付**: 2026-04-17（v2.1.113）
- **ソース**: [Changelog - Claude Code Docs](https://code.claude.com/docs/en/changelog) / [Sandboxing - Claude Code Docs](https://code.claude.com/docs/en/sandboxing) / [GitHub Issue #35159 - anthropics/claude-code](https://github.com/anthropics/claude-code/issues/35159)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: v2.1.113で `sandbox.network.deniedDomains` 設定が追加。`allowedDomains` でワイルドカード（例: `*.company.com`）を許可している場合でも、特定サブドメイン（例: `secrets.company.com`、`vault.company.com`）を明示的にブロックできるようになった。GitHubのIssueでコミュニティから要望があり実装に至った機能で、エンタープライズ環境でのエージェント運用セキュリティを細粒度で制御できる。
- **考察**: エンタープライズ・セキュリティ担当者向けの実用Tips。「Claude Codeに社内の認証情報系エンドポイントを触らせたくない」というニーズへの直接的な回答。Managed AgentsやRoutines（既投稿）と組み合わせた本番運用のセキュリティチェックリストとして活用できる。
- **注目度**: 中
- **推奨投稿型**: Tips型
