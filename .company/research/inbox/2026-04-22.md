### 2. Claude Codeが23年前のLinuxカーネル脆弱性を発見

- **日付**: 2026-04-07（カンファレンス発表） / 4/7以降InfoQ・HN等で拡散
- **ソース**: [Claude Code Used to Find Remotely Exploitable Linux Kernel Vulnerability Hidden for 23 Years — InfoQ](https://www.infoq.com/news/2026/04/claude-code-linux-vulnerability/) / [Adafruit Blog](https://blog.adafruit.com/2026/04/07/claude-code-found-a-linux-vulnerability-that-had-remained-hidden-for-23-years/) / [mtlynch.io](https://mtlynch.io/claude-code-found-linux-vulnerability/) / [Hacker News](https://news.ycombinator.com/item?id=47633855)
- **カテゴリ**: 技術動向
- **事実**: Anthropic研究者のNicholas CarliniがClaude Code（Opus 4.6）を使い、LinuxカーネルのNFSドライバに2003年から存在していたリモート悪用可能なヒープバッファオーバーフローを発見。単純なBashスクリプトでカーネル全ソースファイルをClaude Codeに順番に渡す手法を使用。合計500件以上の高深刻度ゼロデイ脆弱性を検証済み。Opus 4.1やSonnet 4.5ではほとんど再現できなかったことから、能力の急激な跳ね上がりが確認された。
- **考察**: Claude CodeがOSSの安全性を底上げするという具体的・数値的な事例。「AIがセキュリティ研究者の仕事をするようになった」という文脈で速報的に紹介できる。単純なスクリプトで実現できた点が「再現性」を演出し、Tips型にも転用可能。
- **注目度**: 高
- **推奨投稿型**: 速報型

- **考察**: Claude Codeをパワフルに使いたいユーザーへの直接的なコスト影響トピック。「固定費で無制限に使えた黄金時代の終焉」という切り口で共感を得やすい。extra usageへの移行指南をTips的に添えると実用度が上がる。

- **速報型（Linux脆弱性発見）**: 4/7の出来事で直近1週間以内。「23年間誰も気づかなかった脆弱性をClaude Codeが発見」というキャッチーな事実が速報として機能する。HN・InfoQ等で実績ある話題性。

- **Tips型（CLAUDE.md設計）**: Boris Cherny本人の実例という信頼性。「設定を増やせば良いと思っていた」というよくある誤解を正す切り口は実務に直結する。候補5の@メンションTipsとセットにしても面白い。
