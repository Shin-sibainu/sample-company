### 1. Claude Security がパブリックベータ公開 — コードベース脆弱性スキャン＆自動パッチ

- **日付**: 2026-04-30 〜 2026-05-01
- **ソース**: [Anthropic announces Claude Security public beta - SiliconANGLE](https://siliconangle.com/2026/04/30/anthropic-announces-claude-security-public-beta-find-fix-software-vulnerabilities/) / [Claude Security public beta - Help Net Security](https://www.helpnetsecurity.com/2026/05/04/anthropic-claude-security-public-beta/) / [Anthropic - Infosecurity Magazine](https://www.infosecurity-magazine.com/news/anthropic-claude-security-for-ai/)

- **事実**: Anthropicが Claude Security（旧称 Claude Code Security）をエンタープライズ顧客向けパブリックベータとして4月30日に発表。Opus 4.7 を用いてコードベースの脆弱性を検出し、パッチ案を自動生成する。スケジュールスキャン・ターゲットスキャン・ディレクトリ指定・トリアージ追跡・CSV/Markdown エクスポート・Slack/Jira Webhook 連携が可能。既存ツールでは発見できなかった脆弱性を本番コードで検出した組織が数百社に上るという。Team・Max 向けアクセスも後日予定。
- **考察**: Claude Code が「コーディング支援」から「セキュリティ工程の自動化」へ領域を拡大した節目。特にエンタープライズ開発者にとって、既存 SAST ツールとの差別化（自然言語でパッチを生成）が刺さる訴求ポイントになる。「脆弱性を何年も見落としていた既存ツール」という文言は拡散しやすいフック。
- **注目度**: 高
- **推奨投稿型**: 速報型

### 2. `claude project purge` コマンド追加（v2.1.126）— プロジェクト状態を一括削除

- **日付**: 2026-05-01
- **ソース**: [Claude Code 2.1.126 Project Purge - FindSkill.ai](https://findskill.ai/blog/claude-code-2-1-126-project-purge-cleanup/) / [Releases · anthropics/claude-code](https://github.com/anthropics/claude-code/releases) / [Claude Code Updates May 2026 - Releasebot](https://releasebot.io/updates/anthropic/claude-code)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: v2.1.126（5月1日リリース）で `claude project purge [path]` コマンドが追加。トランスクリプト・タスク・ファイル履歴・設定エントリをまとめて削除できる。`--dry-run`、`-y/--yes`、`-i/--interactive`、`--all` の各オプションをサポート。同バージョンで WSL2/SSH 環境での OAuth コード貼り付け対応、Windows PowerShell 7 自動検出強化、`allowManagedDomainsOnly` セキュリティ修正も含まれる。
- **考察**: 長期運用しているプロジェクトで肥大化したローカルデータを一掃できる実用コマンド。「蓄積されたトランスクリプトが重くなってきた」という開発者の痛みに直結。WSL2/SSH 対応強化は Windows ユーザーへのリーチ拡大を示す。
- **注目度**: 中
- **推奨投稿型**: Tips型

### 3. Advisor Tool ベータ — Sonnet/Haiku に Opus の知性を注入してコスト 85% 削減

- **日付**: 2026-04-09（ベータ開始）〜 5月も話題継続
- **ソース**: [The advisor strategy: Give Sonnet an intelligence boost with Opus - claude.com](https://claude.com/blog/the-advisor-strategy) / [Claude Advisor API: Use Opus for 80% Less Money - builder.io](https://www.builder.io/blog/the-claude-advisor-pattern) / [Advisor tool - Claude API Docs](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: Sonnet/Haiku がエグゼキューターとして動作し、判断が困難な場面でのみ Opus をアドバイザーとして呼び出す「Advisor Tool」がパブリックベータ。SWE-bench Multilingual で Sonnet+Opus アドバイザー構成が 74.8%（単独 Sonnet 比 +2.7pt）を達成しつつ、Opus 単体より 11.9% コスト削減。Haiku+Opus アドバイザー構成では単独 Sonnet 比 29pt 低スコアながら 85% コスト削減。API ヘッダー `anthropic-beta: advisor-tool-2026-03-01` を追加するだけで利用可能。
- **考察**: 「高精度 × 低コスト」のトレードオフを動的に解決するアーキテクチャパターンとして実用性が高い。Claude Code SDK との統合が進めば、エージェント設計の定石として広まる可能性がある。コスト削減の具体的数値（85%）はキャッチーで拡散しやすい。
- **注目度**: 高
- **推奨投稿型**: 解説型
