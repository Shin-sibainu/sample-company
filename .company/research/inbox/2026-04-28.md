### 2. Claude Opus 4.7 リリース・Claude Code のデフォルトモデルに採用

- **日付**: 2026-04-16
- **ソース**: [Introducing Claude Opus 4.7 - Anthropic](https://www.anthropic.com/news/claude-opus-4-7)
- **カテゴリ**: 公式発表
- **事実**: Claude Opus 4.7 が GA となり、Max・Team Premiumプランでのデフォルトモデルに設定された。CursorBench で 70%（4.6 比 +12pt）、Rakuten-SWE-Bench で 3 倍の本番タスク解決、内部 93 タスクベンチマークで +13% を達成。新設の `xhigh` 努力レベルがほとんどのコーディング作業の推奨設定となった。
- **考察**: ソフトウェアエンジニアリング特化の精度向上が数値で裏付けられており、Claude Code での実務活用を加速させる。`/effort` スライダーとの組み合わせでユーザーが精度と速度をコントロールできる点も訴求ポイント。
- **注目度**: 高
- **推奨投稿型**: 解説型

### 3. Claude Code 品質問題ポストモーテム（4 月 23 日公開）

- **日付**: 2026-04-23
- **ソース**: [An update on recent Claude Code quality reports - Anthropic Engineering](https://www.anthropic.com/engineering/april-23-postmortem)
- **カテゴリ**: 公式発表
- **事実**: Anthropicは、Claude Code・Claude Agent SDK・Claude Cowork の三カ所で品質低下を引き起こした変更を特定・公表した。主な原因は (1) デフォルト推論努力を high→medium に下げた変更、(2) 思考履歴をドロップするキャッシュバグ、(3) コーディング品質を悪化させた冗長性プロンプト変更。v2.1.116（4 月 20 日）で全修正済み。
- **考察**: 公式がポストモーテムを公開するのは珍しく、透明性重視の姿勢を示す。「デフォルトを下げると気づかれる」という教訓は AI ツールのプロダクト運営論として議論を呼びやすい。
- **注目度**: 高
- **推奨投稿型**: 議論型
