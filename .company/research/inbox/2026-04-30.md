### 1. Anthropic公式ポストモーテム: Claude Codeの品質低下の原因を公開（4/23）

- **日付**: 2026-04-23
- **ソース**: [An update on recent Claude Code quality reports](https://www.anthropic.com/engineering/april-23-postmortem)
- **カテゴリ**: 公式発表
- **事実**: Anthropicが約6週間にわたるClaude Codeのパフォーマンス低下について公式ポストモーテムを公開。原因は3件の重なった変更。(1) 3月4日〜4月7日: デフォルト推論努力レベルをhigh→mediumに変更（レイテンシ削減目的だったが逆効果）、(2) 3月26日〜4月10日: アイドルセッションのキャッシュクリア機能にバグが混入しターンごとに発動、(3) 4月16日〜4月20日: 冗長さ削減の追加システムプロンプト指示がコーディング品質を低下させた。全問題はv2.1.116（4月20日）で解消済み。Anthropicは4月23日時点で全サブスクライバーの使用制限をリセット。
- **考察**: ユーザーの「品質が落ちた気がする」という感覚が事実だったことをAnthropicが認め、3つの具体的な原因を開示した透明性の高い対応。AI企業のインシデント対応の手本として、また「システムプロンプトの変更がモデル品質にどう影響するか」という技術的教訓としても非常に注目度が高い。
- **注目度**: 高
- **推奨投稿型**: 解説型

### 2. Claude CodeのProプランからの一時削除と撤回（4/21〜4/23）

- **日付**: 2026-04-22
- **ソース**: [Anthropic tests reaction to yanking Claude Code from Pro • The Register](https://www.theregister.com/2026/04/22/anthropic_removes_claude_code_pro/)
- **カテゴリ**: 公式発表 / コミュニティ
- **事実**: 4月21日午後、Anthropicが月額$20のProプランの価格ページからClaude Codeのアクセス記載を削除。既存ユーザーへの影響はなかったが、新規ユーザーに対するA/Bテスト（約2%対象）として実施。コミュニティからの強い反発を受け、24時間以内の4月23日に撤回・復元。Anthropic成長責任者は「現行プランはこの使用量には設計されていなかった」と説明し、将来的なプラン再設計を示唆。根本問題はサブスクリプション料金がトークン消費コストの10倍以上安い場合があること。
- **考察**: Anthropicが$20プランの収益性に懸念を持ち始めている証拠。将来的な価格改定の前兆となる可能性があり、独立開発者やインディーハッカーにとって重要なシグナル。「いつまでProプランでClaude Codeが使えるか」という議論を呼ぶ。
- **注目度**: 高
- **推奨投稿型**: 速報型

### 3. Claude Code Routines発表: スケジュール・イベント駆動のクラウド自動化（4/14）

- **日付**: 2026-04-14
- **ソース**: [Introducing routines in Claude Code | Claude](https://claude.com/blog/introducing-routines-in-claude-code)
- **カテゴリ**: 公式発表
- **事実**: Routinesはプロンプト・リポジトリ・コネクターを一度設定すれば、スケジュール・APIコール・GitHubイベント（PR作成、リリース公開等）を契機にクラウド上で自動実行される機能。ローカルPCをオンラインにしておく必要がない。Proユーザーは1日5回、Maxユーザーは1日15回、Team/Enterpriseは1日25回まで実行可能。Claude Code on the webと統合。
- **考察**: 「コーディングAIをCI/CDパイプラインに組み込む」という開発者の要望に直接応えた機能。ローカル実行不要でバックグラウンドエージェントが動くことで、Claude Codeの使い方が「チャットツール」から「自律エージェント基盤」へと大きく変化する転換点。
- **注目度**: 高
- **推奨投稿型**: 解説型

### 4. /ultrareview: 並列マルチエージェントコードレビュー機能（4/16）

- **日付**: 2026-04-16
- **ソース**: [Claude Code /ultrareview: the Bug-Hunting Agent Fleet in the Cloud (April 2026)](https://pasqualepillitteri.it/en/news/1301/claude-code-ultrareview-agents-cloud-2026)
- **カテゴリ**: 公式発表
- **事実**: /ultrareviewはブランチを複数の並列レビュワーエージェントに展開し、各エージェントの発見に対してさらに敵対的批評パスを実行して検証済み知見レポートを返す。CLIでは`claude ultrareview [target]`サブコマンドでCI/スクリプトからも呼び出し可能（--jsonオプションでJSON出力）。検証できないバグは報告されない仕組み。Pro/Maxユーザーは5月5日まで3回無料。
- **考察**: 単一エージェントのレビューと違い、「発見できなければ報告しない」という検証フローが誤検知を減らす。CI組み込み用CLIコマンドの追加もあり、実務での採用ハードルが低い。
- **注目度**: 高
- **推奨投稿型**: Tips型

### 5. Claude Code ソースコード512,000行が誤公開（3/31）

- **日付**: 2026-03-31
- **ソース**: [Claude Code's source code appears to have leaked: here's what we know | VentureBeat](https://venturebeat.com/technology/claude-codes-source-code-appears-to-have-leaked-heres-what-we-know)
- **カテゴリ**: コミュニティ / 技術動向
- **事実**: npmパッケージ @anthropic-ai/claude-code v2.1.88 に59.8 MBのJavaScriptソースマップ（.mapファイル）が誤同梱され、512,000行・1,906のTypeScriptファイルが公開状態になった。Bun（ビルドツール）がデフォルトでソースマップを生成する仕様を利用していたが、`.npmignore`に`*.map`の記載を忘れた人的ミスが原因。内部に44の隠れた機能フラグ、常時バックグラウンドエージェント「KAIROS」、アンダーカバーモード（オープンソースへの匿名コントリビュート機能）等が含まれていた。Anthropic公式コメント: 「顧客データや認証情報は含まれない、リリースパッケージングの人的ミス」。
- **考察**: 1週間前の出来事だが今も技術コミュニティで分析継続中。内部アーキテクチャが明らかになったことで、Claude Codeの実装方針への理解が深まった。Anthropicが取得したBun（後述）のソースマップ生成仕様が直接の引き金という皮肉も注目点。
- **注目度**: 高
- **推奨投稿型**: 議論型
