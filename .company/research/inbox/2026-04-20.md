## 候補一覧

### 1. Claude Opus 4.7 が Claude Code に統合 — xhigh effort & Auto mode

- **日付**: 2026-04-16
- **ソース**: [Introducing Claude Opus 4.7](https://www.anthropic.com/news/claude-opus-4-7) / [Anthropic launches Claude Design following Opus 4.7 model upgrade - 9to5Mac](https://9to5mac.com/2026/04/17/anthropic-launches-claude-design-for-mac-following-opus-4-7-model-upgrade/)
- **カテゴリ**: 公式発表
- **事実**: Anthropic が Claude Opus 4.7 を 4/16 にリリース。ビジョン解像度が 3× 向上（98.5% visual-acuity）、CursorBench で 70%（+12pt）、93タスクコーディングベンチマークで 13% 改善。Claude Code には「xhigh」effortレベルが追加され全プランでデフォルト化。Max ユーザーには Auto mode も解禁。プロ・Maxユーザーには ultrareview 3 回分が無料提供。
- **考察**: モデル刷新 × Claude Code の実装性能向上は開発者に直撃する速報ネタ。「前モデルより本当に速いのか」「コスト変わるのか」など実務目線のコメントも集まりやすい。価格据え置き（入力 $5/M、出力 $25/M）という事実も追記すると有用。
- **注目度**: 高
- **推奨投稿型**: 速報型

### 2. Claude Code on the Web — ブラウザからタスクをクラウド実行

- **日付**: 2026-04-14 ～ 継続中
- **ソース**: [Claude Code on the web | Claude](https://www.anthropic.com/news/claude-code-on-the-web)
- **カテゴリ**: 公式発表
- **事実**: Pro・Max ユーザー向けに「Claude Code on the web（research preview）」が公開。claude.com/code からリポジトリを接続し、ローカル環境不要でコーディングタスクをクラウド実行できる。各タスクはサンドボックス環境で動作し、ネットワーク制限・Gitプロキシで安全性を担保。iOS アプリ対応も含まれ、モバイルからもタスクをアサイン可能。
- **考察**: 「ターミナル不要でブラウザ・スマホからエージェントを走らせる」は非エンジニア層への訴求力が高い。Routines（4/17 既出）と組み合わせると「常時稼働するAI開発者」というコンセプトを補完できる。切り口を「ローカル環境解放」に絞ると差別化できる。
- **注目度**: 高
- **推奨投稿型**: 解説型

### 3. Claude Managed Agents パブリックベータ — エージェントをAPI一本で本番稼働

- **日付**: 2026-04-08
- **ソース**: [Anthropic launches Claude Managed Agents - SiliconANGLE](https://siliconangle.com/2026/04/08/anthropic-launches-claude-managed-agents-speed-ai-agent-development/) / [Medium解説](https://medium.com/ai-software-engineer/anthropic-launches-claude-managed-agents-that-make-agentic-ai-workflows-real-91134b6f2b56)
- **カテゴリ**: 公式発表
- **事実**: 4/8 に Claude Managed Agents のパブリックベータが Claude Platform で正式スタート。「エージェントループ + ツール実行 + サンドボックスコンテナ + 状態永続化」をセットにした REST API を提供。自前インフラを持たなくても本格的な自律エージェントを 5 分で立ち上げられると謳う。Claude Code SDK（→ Claude Agent SDK にリネーム）がその基盤。
- **考察**: 「Claude Code CLI ユーザー」から「自社サービスにエージェントを組み込む開発者」への橋渡しトピック。Claude Codeをアプリに埋め込みたい層に刺さる。ハンズオン系のポストと相性がよく、DEV.to・Medium での反応も既に多い。
- **注目度**: 高
- **推奨投稿型**: 解説型

- **考察**: Claude Code との直接の関係は薄いが、Anthropic の製品展開を語る文脈で重要。「コーディングエージェント → UIデザインまで一気通貫」という流れの証左として議論型ポストに活用できる。Figma 株ショックもフックになる。

### 5. /ultrareview — 並列マルチエージェントでクラウドコードレビュー

- **日付**: 2026-04-16 ～
- **ソース**: [What's new - Claude Code Docs](https://code.claude.com/docs/en/whats-new) / [Claude Code Changelog: All Release Notes 2026](https://claudefa.st/blog/guide/changelog)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: Claude Code に `/ultrareview` コマンドが追加。クラウド上で並列マルチエージェント分析・批評を実行し、包括的なコードレビューを自動生成する。Pro・Maxユーザーには 3 回分の無料トライアルが付与されている。同タイミングで `/effort` コマンドによるエフォートレベルのセッション単位切り替えも実装。
- **考察**: コードレビューの自動化は実務直結。PR 前の品質チェックとして定型化できる。Routines と組み合わせて「PR 作成トリガーで自動 /ultrareview」が可能になるワークフローは Tips 型ポストの格好の素材。
- **注目度**: 高
- **推奨投稿型**: Tips型

### 6. Caveman Mode — トークンを 65% 削減するコミュニティ発スキル

- **日付**: 2026-04-03（Reddit 急上昇）〜 4/18 時点でも話題継続
- **ソース**: [Claude Code Caveman Mode: Save > 75% on Usage](https://www.nathanonn.com/claude-code-caveman-mode/) / [GitHub - JuliusBrussee/caveman](https://github.com/juliusbrussee/caveman) / [Medium解説](https://medium.com/@joe.njenga/i-tested-claude-code-caveman-new-trick-and-it-cuts-token-costs-by-75-ddb142d2be85)
- **カテゴリ**: コミュニティ
- **事実**: Julius Brussee が作成した Claude Code スキル「Caveman」が r/ClaudeAI で爆発的に拡散。「why use many token when few do trick」— 応答から冠詞・フィラーワード・丁寧表現を除いてコードは維持するスキル。実測では出力トークンを 65% 削減。ただし出力トークンは課金全体の 0.6〜2.5% 程度のため、実際の節約は総コストの 4〜5%。CLAUDE.md をケイブマン語に書き直す「Caveman Compress」も存在し、入力トークンを 46% 削減できる。
- **考察**: 「75% 削減」のキャッチーな見出しと実態（4〜5%）のギャップ自体がネタになる。ファクトチェック系・「試してみた」系ポストとして日本語コミュニティへの紹介に最適。コスト節約の話題は常に注目を集めやすい。
- **注目度**: 高
- **推奨投稿型**: Tips型

### 7. Task Budgets — 長時間エージェント実行のコスト暴走を防ぐ新機能

- **日付**: 2026-04-16（パブリックベータ）
- **ソース**: [Claude Opus 4.7: Complete Guide - NxCode](https://www.nxcode.io/resources/news/claude-opus-4-7-complete-guide-features-benchmarks-pricing-2026) / [Releasebot Claude Code](https://releasebot.io/updates/anthropic/claude-code)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: Claude Code の長時間ランに対してトークン消費上限を設定できる「Task Budgets」がパブリックベータとして提供開始。複雑なエージェントタスクが予期せず大量トークンを消費する「コスト暴走」を防ぐ設定。`/config` または設定ファイルから指定可能。
- **考察**: 「Claude Code を怖くて長時間タスクに任せられない」という層への直接的な回答。エンタープライズ・チーム利用でのコスト管理ニーズを満たす機能。Managed Agents や Routines と組み合わせた本格的なエージェント運用を後押しするトピック。
- **注目度**: 中
- **推奨投稿型**: Tips型

### 9. /session recap & prompt caching 1 時間対応 — セッション継続性の強化

- **日付**: 2026-04-14 ～
- **ソース**: [Claude Code Changelog: All Release Notes 2026](https://claudefa.st/blog/guide/changelog) / [Releasebot Claude Code](https://releasebot.io/updates/anthropic/claude-code)
- **カテゴリ**: 公式発表（機能追加）
- **事実**: `/recap` コマンドでセッションサマリーをその場で取得できる機能が追加（`/config` でも設定可）。同時に prompt caching が 1 時間キャッシュ・強制 5 分キャッシュの 2 段階に対応。長時間セッションで同じコンテキストを再送するコストを削減。
- **考察**: 「長いセッションで Claude が文脈を忘れる問題」への公式対処。PreCompact フック（既投稿）と合わせて「コンテキスト管理の進化」として解説すると深みが出る。実務的なメリットが明快で Tips 型に向く。
- **注目度**: 中
- **推奨投稿型**: Tips型

### 10. Ultraplan — クラウドで計画を立てて Web Editor でレビューする新ワークフロー

- **日付**: 2026-04-06 ～（Week 15 early preview）
- **ソース**: [What's new - Claude Code Docs](https://code.claude.com/docs/en/whats-new) / [Claude Code Changelog](https://claudefa.st/blog/guide/changelog)
- **カテゴリ**: 公式発表（プレビュー）
- **事実**: Ultraplan が early preview に入った。CLI でプランをドラフトしてクラウドに送信 → Web Editor でコメント付きレビュー → リモート実行またはローカルに pull back というワークフロー。初回実行時にクラウド環境が自動生成される。既存の Plan Mode をクラウドに拡張した位置づけ。
- **考察**: 「Plan Mode（既投稿）」の進化形として言及できる。「ローカルで計画 → リモートで実行」の非同期性が Routines（既投稿）と似た思想。切り口は「AIと非同期でコラボレーションする新しい開発スタイル」で独自性を出せる。
- **注目度**: 中
- **推奨投稿型**: 解説型
