> ソース: [Claude Code on the web | Claude](https://www.anthropic.com/news/claude-code-on-the-web)
> 推奨投稿時刻: 15:00

Claude Code on the Web、使ってみたけどこれターミナル不要なのが思ってた以上に体験として違う。

ブラウザのclaude.com/codeからリポジトリ接続して、コーディングタスクをクラウドのサンドボックスで実行できる。ネットワーク制限とGitプロキシがかかってるから「野良実行が怖い」という感覚も薄れる。iOSアプリでも動くので、外出先でタスクを投げて帰るまでに終わってた、というのが普通にできる。

最初は「CLIのほうが速いのにわざわざブラウザ？」と思ったんだけど、触ってみるとUIで状態確認しながら非同期でエージェントを走らせるのはそれはそれでやりやすい。チームに非エンジニアがいる場合は特に説明コストが下がりそう。

こういうエージェントの組み方を体系的に整理してるのがここ → https://claude-code-academy.dev/

Pro・Maxユーザーは今なら3回分の無料トライアルがついてるから、試してみて「これ習慣にしたい」と思えばそのまま継続すればいい。「要らなかった」なら無料の3回で終わる話。

Routinesと組み合わせてPRトリガーで自動実行する設定も一応書けるらしいけど、まずは手動から始めるほうが感覚を掴みやすい。

## 投稿4（議論型）【約360文字】
> ソース: [Anthropic launches Claude Design - TechCrunch](https://techcrunch.com/2026/04/17/anthropic-launches-claude-design-a-new-product-for-creating-quick-visuals/)
> 推奨投稿時刻: 03:00

Anthropicがデザインツール「Claude Design」を出したら、発表日にFigma株が7%落ちた。

スライド・プロトタイプ・ワンペーパーをプロンプトから生成して、PDF・PPTX・Canvaでエクスポート。しかもコードベースやデザインファイルを読み込んでスタイルの一貫性まで保つという。

コーディングツールから始まったAnthropicがUIデザインまで手を出してきた。この半年でClaude Codeからここまで守備範囲が広がるとは正直思ってなかった。

自分はデザイナーじゃないからFigmaへの影響は実感として薄いんだけど、「AIが作ったプロトタイプをレビューする仕事」になっていくのか、それとも全然使いものにならなくて杞憂で終わるのか、まあ実際どうなるかはわからんけど。デザイン系の人、どう見てる？
