v2.1.126、地味に実用的なアップデートが詰まってた。

MCPの `alwaysLoad` オプションも追加。サーバー設定で `true` にしておくと、ツール検索の遅延をスキップして常時使える状態になる。よく使うツールを毎回ロード待ちしてたのが地味にストレスだったので助かる。

あとWindowsのPowerShell 7検出も改善されてた。環境依存の挙動に悩んでた人にはいいニュース。

> ソース: [What's new - Claude Code Docs](https://code.claude.com/docs/en/whats-new)
> 推奨投稿時刻: 15:00

`/ultrareview` を実際に動かしてみてわかったんだけど、これ「コードレビューの補助」じゃなくて「自分が気づけないバグを探す専用エージェント群」として使うのが正しい理解だと思う。

仕組みとしては、クラウド上でバグ探し専門のエージェントが並列で動いて、見つけた問題を CLI かデスクトップに返してくれる。自分がレビューしながら「なんか見落としてそう」という感覚を持ち続けるの、正直しんどい。その不安を外に出せるというか。

まだパブリックリサーチプレビューなので完璧ではないけど、マルチエージェントをコードレビューに使うという方向性は個人的にかなり正しいと思ってる。「上級エンジニアにレビューしてもらう」感覚に近い。

Claude Code をもっと実務で使いこなしたい人は → https://claude-code-academy.dev/

## 投稿3（Tips型）【約270文字】
> ソース: [Claude Code Tips I Wish I'd Had From Day One (marmelab.com)](https://marmelab.com/blog/2026/04/24/claude-code-tips-i-wish-id-had-from-day-one.html)
> 推奨投稿時刻: 21:00

複雑な作業を始める前に Shift+Tab を2回押してプランモードに入るの、最初は「めんどくさい」と思って飛ばしてたんだけど、一度やったら戻れなくなった。

実装に入る前に「どうやるか」を先に確認して合意できるので、後から「全然違う方向に進んでた」みたいな事故が減る。特に大きめのリファクタリングや仕様が曖昧なタスクに効く。

あと `/rewind` を知ってから実験のハードルが下がった。「失敗しても会話履歴ごと戻せる」とわかってると、とりあえず試してみるってやりやすくなる。CLAUDE.md にプロジェクトのコンテキストを書いておくのも合わせてやると、毎回の説明コストが消える。

この3つを組み合わせるだけで体感かなり変わるはず。

## 投稿4（議論型）【約280文字】
> ソース: [Anthropic tests reaction to yanking Claude Code from Pro • The Register](https://www.theregister.com/2026/04/22/anthropic_removes_claude_code_pro/)
> 推奨投稿時刻: 03:00

Claude Code の Pro プラン除外騒動、「ユーザーの反発で撤回された」という読み方もできるけど、背景にあるコスト構造の問題は解決してないよな、と思ってる。

サブスクリプション料金とトークン消費量のコスト差が最大10倍になってるという話が出てて、そりゃ Anthropic 側も持続可能かどうか考えるよな、という気持ちもある。かといって告知なしでランディングページごと書き換えるのは信頼を削る。

個人的には「値上げするなら堂々と値上げして理由を説明してほしい」派。コミュニティの反発で一時撤回して、また別のタイミングで試みるという繰り返しになるくらいなら、最初から透明に議論してほしい。

みんなはこの件、どう思った？ #ClaudeCode
