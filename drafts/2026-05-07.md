## 投稿2（解説型・CTA可）【約380文字】
> ソース: [Higher usage limits for Claude and a compute deal with SpaceX — Anthropic](https://www.anthropic.com/news/higher-limits-spacex)
> 推奨投稿時刻: 15:00

Anthropic が SpaceX の Colossus データセンターと契約したやつ、最初ニュース見て「なんで？」ってなったんだけど、規模を調べたら納得感があった。

300MWを超える電力容量、GPU 22万基以上。単純にそこまでのコンピュートを一気に確保できるところが他にほぼなかった、ということなんだろうと思う。

実際の恩恵として、Pro・Max・Team・Enterprise の5時間レート制限が倍増された。ピーク時間帯の制限も完全撤廃。API の Tier 1 は最大入力トークン数が 1500% 増。

「xAI の Colossus と手を組むのはどうなんだ」という声もあるんだけど、自分としては実用上は関係ないかなという感想。どこのインフラ使おうが Claude の挙動が変わるわけじゃないし。それより重い処理を昼間に流せるようになったほうがうれしい。

Claude Code をがっつり使い倒したい人は → https://claude-code-academy.dev/

## 投稿3（Tips型）【約290文字】
> ソース: [Releases · anthropics/claude-code — GitHub](https://github.com/anthropics/claude-code/releases)
> 推奨投稿時刻: 21:00

最近の Claude Code アップデートで地味に便利だなと思ったやつ。

`claude project purge` を使うとプロジェクトに溜まった Claude Code の状態を一括削除できる。別のプロジェクトを長く触ったあとに戻ってきたとき、古い文脈が残ってて微妙な挙動をすることがあって、そういうときに使いたかったコマンド。

あと MCP サーバーが起動エラーのとき最大3回まで自動リトライするようになった。チームで複数人が MCP を使ってると、一時的なエラーで「接続できないです」ってなることがあったんだけど少し落ち着きそう。

`/mcp` コマンドで接続サーバーのツール数も見えるようになったので、確認がラクになった。

## 投稿4（議論型）【約320文字】
> ソース: [Inside Claude Code Auto Mode — InfoQ](https://www.infoq.com/news/2026/05/anthropic-claude-code-auto-mode/) / [Claude Code auto mode: a safer way to skip permissions — Anthropic Engineering](https://www.anthropic.com/engineering/claude-code-auto-mode)
> 推奨投稿時刻: 03:00

Claude Code Auto Mode の技術詳細がブログに出てたんだけど、「3回連続で拒否、または累計20回拒否したら強制的に人間にエスカレーション」という仕様が個人的にはかなり面白いと思った。

AIが自分の判断に迷い続けたら人に戻すっていうのは理にかなってるんだけど、逆に言うと「Claude がエスカレーションしてきたときは本当に曖昧なタスクだった」という指標にも使えるよな、と。

現時点では Team ユーザー向けのリサーチプレビューで、大量ファイル削除・機密データ流出・悪意あるコード実行は自動ブロックする設計らしい。

自律コーディングの安全性、どこまで信頼できると思う？ 実際に使ってる人の感覚を聞きたい。
