## 投稿2（解説型・CTA付き）【約400文字】
> ソース: [Orchestrate teams of Claude Code sessions - Claude Code Docs](https://code.claude.com/docs/en/agent-teams) / [From Tasks to Swarms: Agent Teams in Claude Code - alexop.dev](https://alexop.dev/posts/from-tasks-to-swarms-agent-teams-in-claude-code/)
> 推奨投稿時刻: 15:00

Claude CodeのAgent Teamsを試してみた。環境変数1本で別次元の体験になる。

`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` をセットすると、1つのセッションがチームリードになって複数のClaudeインスタンスを起動して並列作業させられる。サブエージェントと何が違うかというと、チームメイト同士がメールボックス経由で直接やり取りして共有タスクリストで自律協調できる点。「オーケストレーターが全部仕切る」じゃなくて「チームで協議しながら進む」感じに近い。

フロントエンド・バックエンド・テストを別々のインスタンスが同時に見る、みたいな分担が普通にできる。Shift+Downでインスタンスを切り替えながらそれぞれの進捗を確認できるので、「自分がスクラムマスターになってAIチームを監視してる」という感覚が出てくる。

v2.1.114でクラッシュバグが直ったので今が使い始めのタイミング。ただトークンは線形でかさむので、コストとの相談は必要。

Agent系の組み方を体系的に整理したのがここ → https://claude-code-academy.dev/

## 投稿3（Tips型）【約320文字】
> ソース: [Boris Cherny on X](https://x.com/bcherny/status/2039421575422980329) / [Fullscreen rendering - Claude Code Docs](https://code.claude.com/docs/en/fullscreen) / [Claude Code v2.1.97: focus view, live status refresh - Buttondown](https://buttondown.com/claudecode/archive/claude-code-v2197-focus-view-live-status-refresh/)
> 推奨投稿時刻: 21:00

ターミナルのちらつきで集中が切れる人、これだけ覚えておけばいい。

`CLAUDE_CODE_NO_FLICKER=1` を環境変数に追加するとオルタネートスクリーンバッファに切り替わって、長時間セッションでもメモリ・CPU使用量が安定する。自分はマウス操作もオフにしたくて `CLAUDE_CODE_DISABLE_MOUSE=1` と一緒に使ってる。

さらに `/focus`（Ctrl+O）を打つと、プロンプト・ツール要約・最終レスポンスだけ表示するFocus Viewに切り替わる。長時間エージェントを流してるとき、余計なログが全部消えてぐっと集中しやすい。

あと地味に嬉しかったのがWrite toolのdiff計算が60%速くなった件。タブや$記号が多い大きめのファイルを扱うとき、前は書き込み中に詰まる感覚があったんだけどかなり解消された。
