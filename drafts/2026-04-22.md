## 投稿2（解説型）【約350文字】
> ソース: [Claude Code on the web | Claude](https://www.anthropic.com/news/claude-code-on-the-web)
> 推奨投稿時刻: 15:00

Claude Code on the Webで一番変わったのはスマホからタスクを投げられること。

iOSアプリから「このPRの修正やっといて」と投げて、電車乗ってる間にサンドボックス上で実行される。帰宅したら終わってる。これ、ターミナルを常時開けない状況でも開発のコンテキストを切らさないでいられるという意味で、思ってたより用途が広い。

「スマホから開発？」と最初は正直ピンとこなかったんだけど、タスクを「実行する」じゃなくて「アサインする」と思い直したら納得できた。ネットワーク制限とGitプロキシがかかってるからセキュリティ的にも怖くない。

Pro・Max ユーザーで試してない人、claude.com/code から直接繋げる。

## 投稿3（Tips型）【約310文字】
> ソース: [What's new - Claude Code Docs](https://code.claude.com/docs/en/whats-new)
> 推奨投稿時刻: 21:00

`/effort` コマンド、まだ使ってない人に教えておきたい。

セッション単位でエフォートレベルを切り替えられる機能で、Opus 4.7と一緒に追加されてる。雑な調査タスクはlowで流して、設計レビューが始まったらその場でxhighに上げる、というのがコマンド一発でできる。

以前はCLAUDE.mdを書き換えるかflagを叩かないと変えられなかったから、途中で「あ、これ重い処理だ」と気づいたタイミングで即対応できるのは地味に助かる。タスクの性質が途中でわかることってわりと多い。

`/effort xhigh` で上げて `/effort low` で戻す。それだけ。
