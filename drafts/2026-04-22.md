> 推奨投稿時刻: 09:00

Claude Code、公式VS Code拡張がベータで出た。

ターミナル不要。VS Codeのサイドバーパネルからそのまま呼び出して、インラインdiffでリアルタイムに変更確認できる。チェックポイント機能もついてて、Escを2回押すか `/rewind` で任意の状態に即巻き戻せる。

デフォルトモデルもSonnet 4.5になって、ターミナルインターフェースもv2.0に刷新。

正直「CLIで十分じゃん」と思ってたけど、IDEから一歩も出ずに差分を見ながら操作できるのは体験としてかなり違う。特に「ターミナルが怖い」という人を巻き込みやすくなるのは大きい。チームで使うなら選択肢として真剣に考えていいと思う。

## 投稿2（解説型・CTA可）【約430文字】
> ソース: [Claude Code: Workflows and Best Practices 2026](https://smart-webtech.com/blog/claude-code-workflows-and-best-practices/) / [Claude best practices 2026: the complete power user guide](https://www.the-ai-corner.com/p/claude-best-practices-power-user-guide-2026)
> 推奨投稿時刻: 15:00

「なんでClaude Codeがうまく動かないんだ」と感じたとき、自分も最初はモデルのせいにしてた。でも違った。

2026年、「Context Engineering」というコンセプトが海外コミュニティで急速に広まってる。要はプロンプトをうまく書くより、Claudeに渡すコンテキスト全体を設計することのほうがはるかに重要だという話。Anthropicの内部テストでは、ガイドなしで実行すると成功率33%。コンテキストを設計し直すと大幅に改善する。

キーになるパターンは三つ。CLAUDE.mdに「このプロジェクトの構造・ルール・やってはいけないこと」を丁寧に書くこと。大きいタスクはPlan Modeで先に設計させること。並列実行できる作業は3〜5セッションのワークツリーに分けること。

要するに「Claudeが迷わないための地図を先に渡す」感覚。プロンプト職人からコンテキスト設計者へ、という変化はじわじわ来てる。

日本語でこのあたりを体系的に整理してるのはまだ少ない。自分がよく参考にしてるのはここ → https://claude-code-academy.dev/

## 投稿3（Tips型）【約320文字】
> ソース: [What's new - Claude Code Docs](https://code.claude.com/docs/en/changelog)
> 推奨投稿時刻: 21:00

v2.1.116（4/20リリース）で `/resume` が最大67%速くなった。

大型プロジェクトで作業中断して再開するとき、今まで「なんか時間かかってるな…」と待ってたやつ。体感でも明確に変わったと思う。40MB超のセッションだと特に効いてくる。

それより個人的に気に入ったのがthinking spinnerの変化。以前は「止まってるのか動いてるのか分からない」状態だったんだけど、"still thinking" → "thinking more" → "almost done thinking" とインラインで状態が変わるようになった。長時間タスクを回しながら「あ、まだ考えてるのか」と分かるのは地味に安心する。

あと `/doctor` がClaude応答中でも開けるようになった。「なんか変だな」と思ったとき中断なしで確認できる。細かいけど積み重ねで作業が止まる回数が減る。
