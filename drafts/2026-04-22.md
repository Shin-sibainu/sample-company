> ソース: [Hooks reference - Claude Code Docs](https://code.claude.com/docs/en/hooks)
> 推奨投稿時刻: 09:00

Claude Codeのフックイベント、気づいたら26種類になってた。

v2.1.89で`PreToolUse`フックに`defer`が追加されたのが地味にでかい。`-p`モードでツール呼び出し直前に一時停止してペイロードを吐いて終了、あとで`--resume`で再開できる。要は「このBashコマンド、本当に実行していい？」を外側のシステムに判断させる仕組みが公式に入った。

プロンプトで「危険なことはしないで」と書くのとは全然違う。Hooksは決定論的に実行されるから、承認ゲートをCI/CDに組み込んで監査証跡も残せる。

v2.1.114時点で複合コマンド（`ls && git push`とか`FOO=bar git push`）への条件付きマッチングのバグも修正済みなので、細かいルール設定も安定して動くようになってる。

## 投稿4（議論型）【約380文字】
> ソース: [Anthropic admits Claude Code quotas running out too fast - The Register](https://www.theregister.com/2026/03/31/anthropic_claude_code_limits/)
> 推奨投稿時刻: 03:00

Max 20×のウィンドウが19分で枯渇する事例が報告されて、Anthropicが「最優先で調査中」と認めたのが3/23。The Registerが4/13に続報を出した時点でも完全解決のアナウンスはなかった。約4週間、ブログ投稿もメール通知もなし。

原因はプロンプトキャッシュのバグでコストが10〜20倍になっていたこと、セッション再開時に全コンテキストを再処理していたこと、などが重複して発生してたらしい。

自分が気になったのは内容より対応の仕方で、「有料サービスが4週間まともに動かないのに公式の説明が何もない」というのは、正直業務で使う前提だとかなりきつい。

AI ツールへの依存が深まってきたいま、障害時の透明性ってどこまで期待していいものなんだろう。みんなはこういうとき代替手段を用意してる？それとも障害が明けるまで待つ派？
