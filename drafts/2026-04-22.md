## 投稿2（解説型・CTA可）【約420文字】
> ソース: [The Advisor Strategy — Anthropic Blog](https://claude.com/blog/the-advisor-strategy)
> 推奨投稿時刻: 15:00

Advisor Strategy、最初に読んだとき「要するにモデルを2つ使うってこと？」と思ってちょっと舐めてたんだけど、数字を見て考えが変わった。

仕組みはシンプルで、安い executor モデル（Haiku や Sonnet）が長時間タスクを処理しながら、詰まったときだけ Opus に相談する。executor が自律的に動いて、本当に必要なときだけ賢い advisor を呼ぶ設計。

Haiku + Opus advisor の組み合わせで、SWE-bench のスコアが Haiku 単独の 19.7% から 41.2% まで上がる。しかも単独 Sonnet 比でコストは 85% 削減。「Haiku の値段で Opus に近い結果」というのが冗談みたいな話で、実際に API で試してみたらちゃんと動いた。

`advisor-tool-2026-03-01` のベータヘッダーをつければ今日から使える。エージェントのコスト設計に悩んでる人は触る価値がある。

こういう「モデルの役割分担」の組み方をもっと体系的に学びたい人は → https://claude-code-academy.dev/

## 投稿3（Tips型）【約350文字】
> ソース: [Claude Platform Release Notes — April 8, 2026](https://platform.claude.com/docs/en/release-notes/overview)
> 推奨投稿時刻: 21:00

プロンプトや設定を .md ファイルで管理してる人に聞いてほしいんだけど、それ Git で追えてる？

`npm install -g @anthropic/ant` で入る ant CLI、エージェント設定・プロンプト・ツール定義を YAML でバージョン管理できる。Claude Code とネイティブ統合されてて、Managed Agents のセッション管理も ant コマンド一本で完結する。

自分が一番ありがたいと思ったのは「設定を変えたのはいつか」が履歴で追えること。プロンプトをちょっと直したら出力が変わった、でも何を変えたか覚えてない——というのが、コードと同じように diff で見れるようになる。

CLAUDE.md の次のステップとして、設定もコードと同じ扱いにするのは理にかなってると思う。インフラをコードで管理する感覚をそのまま AI エージェントの設定に持ち込む、という発想。

## 投稿4（議論型）【約390文字】
> ソース: [Axios: No company in American history has ever grown like Anthropic](https://www.axios.com/2026/04/13/anthropic-revenue-growth-ai)
> 推奨投稿時刻: 03:00

Anthropic が ARR $30B で OpenAI を抜いた、という報道が先週出てた。2025 年末の $9B から 4 か月で 3 倍以上。Axios が「アメリカ企業史上、これほど急成長した企業はない」と書いてる。

成長の主エンジンが Claude Code と法人向け API というのは、毎日使ってる側からすると「あ、自分も数字を動かしてたのか」という妙な感覚がある。

ただ、正直なところ Run Rate は瞬間最大風速の側面もあって、どこまで実態を映してるかは判断が難しい。$1M 以上支出する法人顧客が 2 か月で 500 社→1,000 社超に倍増、という数字のほうが個人的にはリアルに感じた。

「OpenAI を抜いた」という話は、競争としてどう見るか人によって全然違うと思う。使う側からすれば2社が本気でやり合ってくれるほうがツールは良くなるわけで、まあそれはそれでいい話なんだけど。

Claude Code を使い始めたのはいつ頃？みんなはこの成長をどう見てる？
