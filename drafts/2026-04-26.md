> ソース: [What's new - Claude Code Docs](https://code.claude.com/docs/en/whats-new) / [Claude Code Routines: Anthropic's New Cloud Automation](https://pasqualepillitteri.it/en/news/851/claude-code-routines-cloud-automation-guide)
> 推奨投稿時刻: 09:00

Claude Code、ローカルツールを卒業しつつある。

Ultraplanが早期プレビュー開始。クラウドで計画を下書きして、Webエディタでレビューしてから実行 or ローカルに引き戻す、という流れになった。ターミナルの外に出た感じ。

さらに Claude Code Routines もリサーチプレビュー開始。プロンプトとリポジトリとトリガーを設定しておくと、GitHubイベントや定期実行をフックにAnthropicのクラウド上で自動実行してくれる（1日最大15回）。

GitHub Actionsと何が違うんだって最初思ったけど、コードを書く部分ごとAIに渡せるのが違う。CI/CDの「テスト実行」じゃなくて「実装もお願いします」の自動化。

これ本格化したら、Claude CodeがIDEのプラグインじゃなくて開発インフラそのものになる。

## 投稿2（解説型・CTA可）【約430文字】
> ソース: [An update on recent Claude Code quality reports](https://www.anthropic.com/engineering/april-23-postmortem)
> 推奨投稿時刻: 15:00

「Claudeがネーフィングされた」って1ヶ月ずっと言われてたやつ、Anthropicが公式ポストモーテムを出して全部認めた。

原因は3つの変更が重なってたことで、モデル自体の問題じゃなく製品層のバグだったらしい。

① 3月4日: デフォルト推論努力を「high」から「medium」に変更
② 3月26日: キャッシュ最適化のバグで、毎ターン思考履歴が消去されてた
③ 4月16日: システムプロンプトでツール呼び出し間のテキストを25語以内に制限する指示を追加

これ、③なんか特にひどくて、要は「言葉を減らせ」と指示することでClaude自身の推論を圧迫してた。しかも3件が同時期に重なってたから、原因の特定も遅れた。

全件は v2.1.116（4月20日）で修正済みで、4月23日に全サブスクライバーの使用制限もリセットされてる。

自分がClaude Code使ってて「なんか最近ダメじゃない？」と感じた時期と完全に一致してて、変な感じがした。モデルが劣化したと思いこんでたけど、ただのバグだった。

こういうのを自社で分析して公開してくれるのは、他のAI企業と比べてもかなり誠実だと思う。ただ1ヶ月かかったのは遅い。

Claude Codeの使いこなしについては https://claude-code-academy.dev/ でも継続的に追ってます。

## 投稿3（Tips型）【約350文字】
> ソース: [Claude Code Tips I Wish I'd Had From Day One](https://marmelab.com/blog/2026/04/24/claude-code-tips-i-wish-id-had-from-day-one.html)
> 推奨投稿時刻: 21:00

Claude Code使い始めてすぐコード書かせてた頃、なんかちぐはぐなものができること多かった。CLAUDE.mdをちゃんと育てるようになってから変わった。

最近読んだ実践ガイドに書いてあったことで、特に効いた3つ:

**1. コーディング前にPlan Modeを必ず通す**
コードを書く前に計画を立てさせる。急がば回れだけど、実装の手戻りが体感で半分以下になった。

**2. 失敗パターンをCLAUDE.mdに書く**
「このやり方はやめてください」という記録を蓄積していくと、Claude自身が同じ失敗をしなくなる。プロジェクトごとに自分専用のルールブックを作ってる感覚。

**3. git worktreeで並列セッション**
同じリポジトリで10〜15セッション並列で走らせられる。別ブランチを同時に探索できるので、実験コストが下がる。

あと Context7 プラグインでライブラリのドキュメントをバージョン固定参照するのも地味に大事で、Claude が古い API を自信満々に書いてくることがなくなった。

全部やるのは最初しんどいけど、CLAUDE.mdだけでも今すぐ始める価値はある。

## 投稿4（議論型）【約310文字】
> ソース: [Anthropic tests how devs react to yanking Claude Code from Pro plan](https://www.theregister.com/2026/04/22/anthropic_removes_claude_code_pro/) / [UPDATED] News: Anthropic (Briefly) Removes Claude Code From Pro Plan](https://www.wheresyoured.at/news-anthropic-removes-pro-cc/)
> 推奨投稿時刻: 03:00

先週、AnthropicがProプラン（月$20）の価格ページからClaude Codeのチェックマークをこっそり外して、数時間で元に戻したやつ。「新規登録者の2%向けA/Bテスト」という説明だったけど、ランディングページとドキュメントが全体向けに書き換えられてたので普通に混乱した。

個人的には、こういう「A/Bテスト」という言葉の使われ方がずっと気になってる。実験対象を明示せずに変更して、反発が大きければ「テストでした」と撤退できる余地を残すやり方。

背景にはMaxプランがClaude Codeリリース前の設計で、今の使用量に対応しきれてないという構造問題があるらしく、それ自体は理解できる。だけど、その課題をユーザー側に押し付ける前にもっとやることがあるんじゃないかとは思った。

この件をきっかけにローカルモデル（LlamaやMistral）への移行議論が盛り上がったのも興味深い。有料AI依存リスクの話として。

みんなはどう感じた? 「実験台にされた感」vs「まあ企業としては普通」どっちに近い?
