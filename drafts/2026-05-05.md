## 投稿2（解説型・CTA可）【約350文字】
> ソース: [Enabling Claude Code to work more autonomously - Anthropic](https://www.anthropic.com/news/enabling-claude-code-to-work-more-autonomously)
> 推奨投稿時刻: 15:00

Claude Code が今週、自律稼働に向けて三段階進化した。

一つ目は VS Code 拡張の正式 GA。インラインdiff・`@`メンション・スラッシュコマンドが使えるようになって、ターミナルを開かなくても Claude Code を呼べる。IDE から離れたくない派にはかなり大きい変化だと思う。

二つ目は Auto mode。安全だと判断した操作は承認なしで通過させて、リスクのある操作だけブロックする分類器が入った。「毎回 y を押す疲労」がかなり減る。

三つ目は Week 18 の細かい強化で、MCP の `alwaysLoad` オプション・`/skills` への検索ボックス・PostToolUse フックが全ツールの出力を置換可能に・MCP 起動時の自動リトライ 3 回、あたりが追加された。

正直 Auto mode が一番効いてる。判断を委ねるのが怖かったけど、分類器がちゃんと仕事してると体感できると安心感が全然違う。

Claude Code の自律活用をもっと深掘りしたい人は → https://claude-code-academy.dev/

## 投稿3（Tips型）【約260文字】
> ソース: [Enabling Claude Code to work more autonomously - Anthropic](https://www.anthropic.com/news/enabling-claude-code-to-work-more-autonomously)
> 推奨投稿時刻: 21:00

Claude Code を長時間走らせて「あ、方向性ずれてた」となるの、自分もよくやる。

Checkpoints はその保険になる。変更を加える前に自動でスナップショットが保存されて、30日間残る。巻き戻したいときは `Esc` を2回押すか `/rewind` を叩くだけ。

ただ、ここで覚えておくと便利なのが「会話のみ・コードのみ・両方」の3モード選択。コードは正しくて会話だけ迷走した、みたいな状況でも選んで戻れる。

自分は最近「とりあえず走らせてみて、変だったら /rewind」という使い方をするようになってから、Claude Code に任せる量が体感で 1.5 倍くらいになった気がする。
