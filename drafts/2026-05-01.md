## 投稿2（解説型・CTA可）【約280文字】
> ソース: [SiliconANGLE: Anthropic's Claude Code gets automated 'routines' and a desktop makeover](https://siliconangle.com/2026/04/14/anthropics-claude-code-gets-automated-routines-desktop-makeover/)
> 推奨投稿時刻: 15:00

Claude Code の Routines 機能、最初に説明読んだときピンとこなかったけど、使い方がわかってきたらかなりデカかった。

プロンプト・リポジトリ・コネクタをセットにして「パッケージ化」しておくと、スケジュール・GitHub Event・APIコールのどれかをトリガーに Anthropic のクラウド上で自動実行される。ノートPCを閉じたままでも動き続ける。

これって実質 CI/CD の「コードを書かないやつ」版じゃないかと思ってて。GitHub Actions を組まなくても、Claude に指示を入れておくだけでルーティンワークが流れる。

エージェントを "常時待機させる" という発想に慣れるまで少し時間かかったけど、慣れると手放せない。 #ClaudeCode

## 投稿3（Tips型）【約270文字】
> ソース: [Claude Code Tips I Wish I'd Had From Day One - marmelab](https://marmelab.com/blog/2026/04/24/claude-code-tips-i-wish-id-had-from-day-one.html)
> 推奨投稿時刻: 21:00

CLAUDE.md、最初は「なんでも書いとけばいいか」ってどんどん膨らませてたんだけど、100行を超えたあたりからClaude が全部読んでるのか怪しくなってきた感覚があった。

marmelab のブログに「100行以内に絞る」って明示されてて、確かにそうだよなと。あれもこれも書くより、本当に大事なルールだけ残す方が効くらしい。

もうひとつ試したいのが `lessons.md`。ミスが起きたらその場でルール化してファイルに追記していく手法で、プロジェクトが育つにつれてClaude の精度も上がっていく設計になってる。「怒ってるうちにルールにする」的な発想、自分に合ってると思う。

git worktree で並列セッション10〜15本という話も出てたけど、そこはまだ怖くて試してない。 #ClaudeCode

> 推奨投稿時刻: 03:00

Anthropic が4月23日に Claude Code の品質低下に関する Postmortem を公開した件、読んでてちょっと複雑な気持ちになった。

原因は3月末〜4月にかけて独立した3つのエンジニアリング変更が重なったことで、うち一つは「verbosity を下げるためのシステムプロンプト追加」がそのままコーディング品質の悪化につながったというもの。約6週間、ずっとモデルのせいだと思ってた人も多かったと思う。

率直な Postmortem を出したことは評価してる。ただ、初期対応で「ユーザー側の使い方の問題」を示唆する発言があったのは正直引っかかった。品質が下がってると感じたとき「自分の使い方が悪いのかも」と疑ってしまうのは、AIツールならではのしんどさだよなと思う。

みんなはあの6週間、何か感じてた？ #ClaudeCode
