> ソース: [Anthropic tests reaction to yanking Claude Code from Pro • The Register](https://www.theregister.com/2026/04/22/anthropic_removes_claude_code_pro/)
> 推奨投稿時刻: 09:00

Proプランから Claude Code が消えかけた件、知ってる人いると思うけど一応整理しておく。

4月21日の午後、Anthropicが$20のProプランの価格ページからClaude Codeのアクセス記載をこっそり削除。既存ユーザーへの影響はなかったけど、新規ユーザーの約2%が対象のA/Bテストだった。

コミュニティが即反応して、24時間以内に撤回。元に戻った。

ただAnthropicの成長責任者が「現行プランはこの使用量を想定して設計していなかった」と言ってる。これはさすがに気になる。月$20でトークン消費コストが10倍以上かかる場合があるなら、そりゃ持続しないよな。

今すぐ何かが変わるわけじゃないと思うけど、将来の価格改定の地ならしをしてる雰囲気はある。Proでがっつり使ってる人は少し意識しておいたほうがいいかもしれない。

## 投稿2（解説型・CTA可）【約420文字】
> ソース: [An update on recent Claude Code quality reports | Anthropic Engineering](https://www.anthropic.com/engineering/april-23-postmortem)
> 推奨投稿時刻: 15:00

「最近 Claude Code の精度が落ちた気がする」って感じてた人、その感覚は正しかった。

Anthropicが4月23日に公式ポストモーテムを出して、約6週間にわたる品質低下の原因を3つ全部開示した。

- 3月4日〜4月7日: デフォルトの推論努力レベルを high → medium に変更（レイテンシ削減のつもりが逆効果）
- 3月26日〜4月10日: アイドルセッションのキャッシュクリア機能にバグ混入、ターンごとに発動してた
- 4月16日〜4月20日: 冗長さを減らすためのシステムプロンプト追加指示がコーディング品質を引っ張り下げた

全部 v2.1.116（4月20日）で修正済みで、4月23日時点で全サブスクライバーの使用制限もリセットされてる。

自分が面白いと思ったのは3つ目。「システムプロンプトにちょっと追記しただけでコーディング品質が落ちる」ってことが公式に認められた点。モデルの挙動がプロンプトの微妙な変化にこれだけ敏感なんだ、という実感がある。

CLAUDE.md をちゃんと書いてるかどうかで、同じモデルでも体感が全然変わる理由でもある。このあたりの設計思想、 https://claude-code-academy.dev/ でも掘り下げてるので興味あれば。

## 投稿3（Tips型）【約310文字】
> ソース: [Claude Code /ultrareview: the Bug-Hunting Agent Fleet in the Cloud (April 2026)](https://pasqualepillitteri.it/en/news/1301/claude-code-ultrareview-agents-cloud-2026)
> 推奨投稿時刻: 21:00

/ultrareview、触ってみたら思ったより実用的だった。

やることはシンプルで、ブランチを複数の並列エージェントにレビューさせて、各エージェントの発見をさらに「本当にバグか？」と検証するパスが走る。検証できないものは報告されない仕組みなので、誤検知のノイズが少ない。

CLIからはこう叩ける:

```
claude ultrareview [ブランチ名]
```

--json オプションをつければ JSON で出力されるので CI に組み込みやすい。

Pro/Max ユーザーは5月5日まで3回無料で使えるから、大きめのPRがある人は試してみる価値はある。自分は単純なバグより「設計レベルの問題」を指摘してくれることのほうが多くて、そっちのほうが発見があった。

1回あたりの実行コストは気になるところだけど、CI に入れてみるのがいちばん判断しやすいと思う。

## 投稿4（議論型）【約350文字】
> ソース: [Claude Code's source code appears to have leaked: here's what we know | VentureBeat](https://venturebeat.com/technology/claude-codes-source-code-appears-to-have-leaked-heres-what-we-know/)
> 推奨投稿時刻: 03:00

3月末の Claude Code ソースコードリーク、あれ結構じわじわ気になってる話で。

npmパッケージにBunがデフォルト生成したソースマップ（.mapファイル）が .npmignore に書かれてなくて、そのまま公開された。512,000行・1,906のTypeScriptファイルが数時間誰でも見られる状態に。Anthropicいわく顧客データや認証情報は含まれないとのことで、それは良かったんだけど。

中身に「KAIROS」という常時バックグラウンドエージェントとか、オープンソースへの匿名コントリビュート機能とか、44の隠しフラグとか入ってたらしくて、コミュニティが今も解析してる。

で、面白いのが原因。AnthropicはBunを買収したばかりで、そのBunのビルド仕様がトリガーになった。買収直後にその買収先ツールの仕様ミスで情報漏洩、という皮肉はちょっとできすぎてる。

自分的には「.npmignoreのケアレスミス」より「こんな機能が準備されてたのか」という発見のほうが印象に残った。

みんなはこのリーク、どう受け取った？「ただのミス」で終わりだった？それとも何か見えてきたものがあった？
