## 投稿2（解説型・CTA可）【約310文字】
> ソース: [The advisor strategy: Give Sonnet an intelligence boost with Opus](https://claude.com/blog/the-advisor-strategy)
> 推奨投稿時刻: 15:00

Anthropic が出した Advisor Tool、面白いアーキテクチャだなと思って試してた。

仕組みはシンプルで、Sonnet か Haiku がメインで動いて、「これは難しい判断だ」というときだけ Opus を呼び出す構成。常時 Opus を使うより賢く・安くなる、という考え方。

数字で言うと、SWE-bench Multilingual で Sonnet+Opus アドバイザー構成が 74.8%（Sonnet 単独より +2.7pt）を達成しながら、Opus 単体よりコストが 11.9% 安い。Haiku+Opus にすると精度はそれなりに落ちるけど 85% コスト削減。

なにより実装がヘッダー一行追加で済む。`anthropic-beta: advisor-tool-2026-03-01` を付けるだけ。「いつも Opus 一択」な人にとっては発想の転換になる使い方かもしれない。

エージェントの設計パターンをもっと深掘りしたい人は → https://claude-code-academy.dev/

## 投稿4（議論型）【約270文字】
> ソース: [free-claude-code proxy - knightli.com](https://www.knightli.com/en/2026/05/01/free-claude-code-anthropic-compatible-proxy/)
> 推奨投稿時刻: 03:00

「Free Claude Code」というプロキシが 14 週で 20,900 スターを集めて、別の claude-code-router は 33,500 スターを記録してる。

仕組みは Claude Code の API 呼び出しを横取りして、OpenRouter や DeepSeek、Ollama に流すというもの。4 月に Anthropic が Pro/Max のサードパーティエージェント利用を制限したのがトリガーで急成長したらしい。

利用規約的にどうなんという話はあるし、個人的には公式ルートで使う方が安心ではあるんだけど、「本家リポジトリより外部プロジェクトの方がスターが多い」という事実は素直にすごいなと思う。価格や制限への不満がここまでコミュニティの自助努力に変換される、というのは AI ツール業界ではわりと珍しいことじゃないか。

みんなはこういうプロキシ系、実際に使ってる？
