# X自動投稿スキル — Claude Code 情報発信

Claude Codeに関する海外の最新情報を収集し、日本語のX投稿として下書き・投稿するスキル。1日4投稿を想定。

## ワークフロー概要

```
情報収集 → コンテンツ生成（4投稿） → 下書き出力 → X MCP Server で投稿（create_tweet）
                                                 ↑
                                         Claude Code の Routines が
                                         1日4回スキルを自動起動
```

投稿は **X MCP Server (DataWhisker/x-mcp-server)** の `create_tweet` ツール経由。定期実行は **Claude Code Desktop の Routines**（2026-04-14発表）に任せる。GitHub Actions は不使用。

---

## STEP 0: 日付確認

**最初に今日の日付を必ず確認する。** システムから提供される現在日時を使い、情報収集の期間（直近1週間）を正確に判定する。

---

## STEP 1: 情報収集

Claude Code関連の最新情報を **WebSearch** / **WebFetch** で収集する。

### 検索クエリ例

```
"Claude Code" news {今日の日付}
"Claude Code" release {今月}
"Claude Code" update announcement site:anthropic.com
"Claude Code" site:github.com/anthropics releases
"Claude Code" tips OR tricks OR workflow {最近}
"Claude Code" site:reddit.com
```

### 情報ソース優先順位

1. **Anthropic公式** — anthropic.com/blog, code.claude.com/docs（最も信頼性が高い）
2. **GitHub** — anthropics/claude-code のリリースノート・Issues・PR
3. **X海外アカウント** — @AnthropicAI, @alexalbert__ 等の発信
4. **コミュニティ** — Reddit r/ClaudeAI, Hacker News, Medium, Dev.to の議論

### 収集のコツ

- **直近1週間以内（7日）の情報のみ使用。さらにその中で「最新」を優先**。今日の日付から逆算して、新しいものから順に評価する
- 3日以内の情報があればそちらを優先、1週間前後のものは話題性・独自性があるもののみ採用
- 同じネタが複数ソースで言及されていれば「話題になっている」と判断して優先
- 公式発表 > コミュニティの発見 > 個人の意見、の順で信頼度を判断
- 最低7件以上の候補を集めてから、投稿にする4件を選定
- **過去の下書きとのネタ被りを防ぐ**: `drafts/` ディレクトリ内の過去ファイルを確認し、同じトピックを再度取り上げない。4件の投稿間でも同じネタを使わない

## STEP 2: コンテンツ生成

収集した情報から **1日4投稿** を生成する。以下の4つの型を1つずつ使う。

### 投稿の型

#### ① 速報型（ニュース速報）
最新の公式発表やアップデート情報を速報として伝える。

**構成:** 冒頭フック → 何が変わったか → なぜ重要か → 補足情報

#### ② 解説型（深掘り・CTAあり）
海外で話題のトピックを深掘りして解説。**この型のみCTAを含めうる。**

**構成:** フック → 背景説明 → 具体的な内容 → CTA（自然な場合のみ）

CTAリンク: `https://claude-code-academy.dev/`

#### ③ Tips型（実用テクニック）
すぐに使えるTipsやテクニックを紹介。

**構成:** 「知ってた？」系のフック → 具体的な操作 → 効果・ユースケース

#### ④ 議論型（コミュニティ投げかけ・エンゲージメント）
フォロワーとの対話を促す投稿。疑問形で締める。いいね・リプライを誘発する。

**構成:** 自分の意見や観察 → 疑問投げかけ → 呼びかけ

**例:**
```
Claude Code使ってる人に聞きたい。最近のアップデートで一番助かった機能って何？
個人的にはPlan Modeかな。実装前に方向性を決められるから、デカい変更でも安心して進められる。
みんなは？
```

### 生成ルール

- **文字数**: 1投稿あたり**130〜140字を目標**。絶対上限は140字（全角）。URLは23文字としてカウント（X側でt.coに短縮される）。140字に収まらない場合は情報を削って短くする。130字未満の投稿は情報量が少ないので、もう少し具体情報（数値・機能名・具体例）を追加して膨らませる
- **トーン**: カジュアルな日本語。「〜だよ」「〜するやつ」「めちゃくちゃ」等OK。ただし砕けすぎない
- **ハッシュタグ**: 投稿本文の最後には付けない。付ける場合は本文中に自然に1つだけ
- **CTA**: 解説型のみ、かつ**投稿内容とAcademyの関連が自然な場合のみ**含める。「学び方」「使い方の解説」「入門」的な文脈ならCTAを入れる。事件・ゴシップ・速報的な内容にはCTAを入れない。CTAなしの日があっても構わない
- **絵文字**: 必要最低限。Tips型の冒頭に1つ程度、議論型にも1つ程度OK。速報型・解説型には基本不要
- **情報の正確性**: 収集した原文の事実を歪めない。不確かな情報は「〜らしい」「〜とのこと」と明示

## STEP 3: 下書き出力

### 出力先

プロジェクトルートの `drafts/` ディレクトリに日付ファイルで出力する。

```
drafts/
  2026-04-14.md
  2026-04-15.md
  ...
```

### ファイルフォーマット

```markdown
# X投稿下書き — YYYY-MM-DD

## 投稿1（速報型）【XX文字】
> ソース: [記事タイトル](URL)
> 推奨投稿時刻: 09:00

（投稿本文）

---

## 投稿2（解説型・CTA可）【XX文字】
> ソース: [記事タイトル](URL)
> 推奨投稿時刻: 15:00

（投稿本文）

---

## 投稿3（Tips型）【XX文字】
> ソース: [記事タイトル](URL)
> 推奨投稿時刻: 21:00

（投稿本文）

---

## 投稿4（議論型）【XX文字】
> ソース: [記事タイトル](URL)
> 推奨投稿時刻: 03:00

（投稿本文）

---

## メモ
- 投稿順: 朝→速報、昼→解説、夜→Tips、深夜→議論
- 各投稿の特徴や選定理由を簡単にメモ

## 投稿ログ（投稿成功時にスキルが書き換える）
- 投稿1: 未投稿
- 投稿2: 未投稿
- 投稿3: 未投稿
- 投稿4: 未投稿
```

### 出力後のアクション

下書きファイルを出力したら、ユーザーに以下を伝える:
- ファイルパスの案内
- 4投稿の簡単なサマリー（各1行、文字数付き）
- 投稿スケジュール（推奨時刻 09:00 / 15:00 / 21:00 / 03:00）
- **X MCP Server の接続確認**: `mcp__x__create_tweet` が利用可能かチェック
  - 利用可能 → 「1件目（速報型）を今すぐ投稿しますか？ 残り3件は推奨時刻に合わせて後で投稿できます」と確認
  - 利用不可 → 「X MCP Server (`x`) が未接続のようです。`.mcp.json` に登録するか、Claude Desktop 側で接続してください」と案内し、投稿はスキップ

## STEP 4: X MCP Server で投稿

**使用サーバー**: [DataWhisker/x-mcp-server](https://github.com/DataWhisker/x-mcp-server)
**使用ツール**: `mcp__x__create_tweet`（引数: `text` に投稿本文）

### 前提（環境側で1回だけ設定）

MCPサーバーの登録と認証キーはスキルの外側で設定する。スキルは「`mcp__x__create_tweet` が呼べる前提」で動く。

**`.mcp.json` への登録例:**
```json
{
  "mcpServers": {
    "x": {
      "command": "node",
      "args": ["/絶対パス/x-mcp-server/build/index.js"],
      "env": {
        "TWITTER_API_KEY": "...",
        "TWITTER_API_SECRET": "...",
        "TWITTER_ACCESS_TOKEN": "...",
        "TWITTER_ACCESS_SECRET": "..."
      }
    }
  }
}
```

**必要な認証情報（OAuth 1.0a・全て必須）:**
- `TWITTER_API_KEY` / `TWITTER_API_SECRET`
- `TWITTER_ACCESS_TOKEN` / `TWITTER_ACCESS_SECRET`

メディア付き投稿を使う場合のみ OAuth 2.0 系の追加キーが必要だが、このスキルは**テキストのみ**なので不要。

### 投稿フロー

1. 下書きファイルから投稿Nの本文を読み取る
2. ユーザーに「投稿N（XX文字）を投稿します。本文: ～ でよろしいですか？」と確認
3. `mcp__x__create_tweet({ text: "..." })` を呼ぶ
4. 返ってきた **tweet ID** を記録
5. 下書きファイルの「投稿ログ」該当行を `✅ 投稿済み (ID: <返却ID>)` に書き換える（Editツール使用）

### 複数投稿のルール（Bot判定回避）

- **連続投稿しない**。デフォルトは「1件目を投稿→残りは推奨時刻まで待つ」
- ユーザーが「まとめて投稿して」と言った場合でも、**最低でも10分以上の間隔**を空ける
- 4件を「今すぐ全部」は拒否する（X側でレート制限 or スパム判定のリスクあり）

### エラー・未接続時の挙動

- `mcp__x__create_tweet` が呼び出せない（未接続）→ 投稿をスキップし、ユーザーに「X MCP Serverが未接続。`.mcp.json` を確認して」と伝える
- 401/403 等の認証エラー → 認証情報の再確認を促す
- 429（レート制限）→ 残り件数と次回可能時刻を伝え、投稿中断
- DataWhisker/x-mcp-server の無料枠上限: **書き込み 約500回/月**。1日4件 × 30日 = 120回なので通常は問題なし

### スケジュール実行は Claude Code の **Routines** で回す

このスキルの定期実行は、Claude Code Desktop (2026-04-14 発表) の **Routines** に載せるのが公式の答え。Routines は保存したプロンプト・リポジトリ・コネクタを、スケジュール / API / GitHub イベントで自動起動できる仕組み。Claude Code の web インフラ上で動くので、ユーザーのPCが落ちていても走る。

#### 2モード構成

このスキルは起動モードを自動判定する:

- **インタラクティブモード**: 人間がチャットで起動した場合。通常通り STEP 1–3 で下書き生成 → ユーザー確認 → 投稿
- **Routinesモード**: Routines からプロンプトで起動された場合。以下の単発動作だけ行う:
  1. `drafts/{today}.md` が存在しない → STEP 1–3 で生成（この時点では投稿しない）
  2. 存在する → 最上位の `未投稿` エントリを1件だけ `mcp__x__create_tweet` で投稿
  3. 該当行を `✅ 投稿済み (ID: <返却ID>)` に書き換え
  4. `git add drafts/ && git commit && git push`

判定は「現在の対話にユーザーからの自然文が含まれているか」で行う。Routinesからの起動はスキル名直指定か定型プロンプトなので、そうでない場合はインタラクティブ扱い。

#### Routines 側の推奨設定

- **プロンプト** (Routine に登録する文):
  ```
  Run the x-auto-post skill in Routines mode: if today's draft file doesn't
  exist in drafts/, generate it first (4 posts) without posting; otherwise
  post the next 未投稿 entry via mcp__x__create_tweet, mark it 投稿済み with
  the returned tweet ID, and commit+push to the current branch.
  ```
- **リポジトリ**: `shin-sibainu/sample-company`
- **コネクタ**: x (DataWhisker/x-mcp-server), GitHub
- **トリガー（Scheduled）**: `0 0,6,12,18 * * *` UTC = JST 09:00 / 15:00 / 21:00 / 03:00 の4回
  - 1日の1回目（JST 09:00 相当）は下書き生成 + 投稿1件
  - 2〜4回目は残りの投稿を順次
- **実行回数**: Pro プランは5回/日が上限。本スキルは4回/日なので枠内。Max 以上はもっと余裕

#### 初期セットアップ（1回だけ）

1. **APIキーを再発行**（チャットに漏れたキーは破棄）
2. Claude Code Desktop → Settings → Connectors で **x (DataWhisker/x-mcp-server)** を登録し、4つの環境変数を入れる
3. Claude Code Desktop → Routines → 新規作成
4. 上記プロンプト・リポジトリ・コネクタ・スケジュールを設定して保存
5. 「今すぐ1回テスト実行」で動作確認

以降はユーザーの関与なし。Routines が勝手に回す。

## 全自動モードへの切替

ユーザーが「全自動にして」と言ったら、STEP 3の確認プロセスをスキップし、下書き生成後に1件目を即 `create_tweet` で投稿する。ただし以下のセーフガードを必ず守る:

- CTA付き投稿が連続しないようチェック
- 同じネタの重複投稿を防止（過去の下書きファイルと照合）
- 4件を一度にまとめ投げしない（最低10分、推奨は各推奨時刻まで待機）
- 投稿ごとに下書きファイルのログ行を `✅ 投稿済み (ID: ...)` に書き換え
- 認証/レート制限エラーは即停止してユーザーに通知
